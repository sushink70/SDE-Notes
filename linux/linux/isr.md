## 1. Fundamentals: What Is an Interrupt?

### 1.1 Definition and Purpose

An **Interrupt** is an asynchronous signal sent to the processor that causes it to suspend its current execution, save its state, and transfer control to a special routine called the **Interrupt Service Routine (ISR)** — also known as an **interrupt handler**. Once the ISR completes its work, the processor restores its saved state and resumes the previously interrupted task.

The key insight is *asynchrony*: a peripheral device (network card, disk controller, keyboard, timer) does not know when the CPU will be ready. Rather than polling devices continuously (which wastes CPU cycles), the CPU is designed to respond to events on demand. This is the interrupt model.

### 1.2 Types of Interrupts

```
Interrupts
├── Hardware Interrupts (Asynchronous)
│   ├── Maskable Interrupts (IRQ)     — Can be disabled via CLI instruction
│   └── Non-Maskable Interrupts (NMI) — Always delivered (hardware faults, watchdog)
│
└── Software Interrupts (Synchronous — also called Exceptions)
    ├── Faults    — Recoverable (page fault, segment fault)
    ├── Traps     — Deliberate (system calls via INT 0x80 / SYSCALL)
    └── Aborts    — Unrecoverable (double fault, machine check)
```

**Hardware Interrupts** are triggered by external devices asserting an interrupt line connected to the CPU's interrupt controller. They are *asynchronous* — they can arrive at any point in the instruction stream.

**Exceptions (Software Interrupts)** are synchronous and generated by the CPU itself in response to executing a specific instruction or encountering a fault condition (divide by zero, invalid opcode, page fault, etc.).

### 1.3 Interrupt vs. Polling vs. DMA

| Mechanism | Description | CPU Usage | Latency | Use Case |
|-----------|-------------|-----------|---------|----------|
| **Polling** | CPU continuously checks device status register | High (busy-wait) | Low | Ultra-low latency I/O (DPDK, SPDK) |
| **Interrupt** | Device notifies CPU when data ready | Low (idle between events) | Medium | General I/O |
| **DMA + Interrupt** | DMA engine moves data; interrupt fires on completion | Minimal | Transfer-time + IRQ | High-throughput bulk I/O |

### 1.4 The Interrupt Lifecycle (Step by Step)

```
1. Device asserts interrupt line (or MSI write to memory)
2. Interrupt controller receives request, arbitrates priority
3. Controller signals CPU via INTR pin
4. CPU finishes current instruction (not mid-instruction)
5. CPU checks IF (Interrupt Flag) — if cleared, defers maskable IRQs
6. CPU saves current execution context (pushed onto stack):
     - RIP  (return address)
     - CS   (code segment)
     - RFLAGS
     - RSP  (stack pointer — if privilege level changes)
     - SS   (stack segment — if privilege level changes)
7. CPU reads interrupt vector from controller (INTA cycle)
8. CPU indexes into IDT using vector number
9. CPU loads new RIP, CS from IDT gate descriptor
10. CPU switches to kernel stack (if coming from user space)
11. ISR begins executing in kernel context
12. ISR services the device (reads status register, clears interrupt flag on device)
13. ISR returns via IRET instruction
14. CPU restores saved context
15. Execution resumes where it was interrupted
```

## 2. Interrupt Hardware Architecture

### 2.1 x86-64 Interrupt Mechanism

On x86-64, the CPU has two relevant pins for interrupts:
- **INTR**: Maskable interrupt input. When `IF=1` in `RFLAGS`, the CPU accepts these.
- **NMI**: Non-Maskable Interrupt input. Always accepted regardless of `IF`.

The `CLI` (Clear Interrupt Flag) and `STI` (Set Interrupt Flag) instructions control whether maskable interrupts are acknowledged. Linux uses these implicitly through spinlock operations (`spin_lock_irqsave` → effectively `CLI`; `spin_unlock_irqrestore` → `STI` if previously enabled).

#### CPU State Saved on Interrupt Entry

When an interrupt causes a privilege-level change (user → kernel), the CPU automatically pushes:

```
Higher addresses (old RSP)
+------------------+
|       SS         |  Stack Segment (saved only on privilege change)
+------------------+
|       RSP        |  User-mode stack pointer
+------------------+
|     RFLAGS       |  Flags (contains IF bit)
+------------------+
|       CS         |  Code Segment
+------------------+
|       RIP        |  Return address (next instruction to resume)
+------------------+  <-- new RSP after CPU push
|   Error Code     |  (only for some exceptions, e.g., page fault)
+------------------+
```

The interrupt entry assembly in Linux (`arch/x86/entry/entry_64.S`) then saves all remaining general-purpose registers and segment registers onto the kernel stack to form a complete `struct pt_regs`.

#### struct pt_regs (x86-64)

```c
struct pt_regs {
    unsigned long r15, r14, r13, r12;
    unsigned long rbp, rbx;
    unsigned long r11, r10, r9, r8;
    unsigned long rax, rcx, rdx, rsi, rdi;
    unsigned long orig_rax;      /* syscall number / error code */
    unsigned long rip, cs, eflags;
    unsigned long rsp, ss;
};
```

This structure is available to ISRs and exception handlers and is also what `ptrace` uses for debugging.

### 2.2 Interrupt Priority and Preemption

On x86, hardware interrupts do **not** automatically preempt each other unless explicitly designed to do so. The `IF` flag is cleared on interrupt entry (by the CPU implicitly when jumping through a gate with `IF=0` semantics). This means:

- By default, an ISR runs with interrupts disabled.
- Linux ISRs can re-enable interrupts for themselves using `local_irq_enable()` but this is discouraged.
- The APIC supports an **IRQ priority** mechanism via the Task Priority Register (TPR) and Processor Priority Register (PPR), which allows higher-priority interrupts to preempt lower-priority ISRs.

### 2.3 ARM GIC (Generic Interrupt Controller)

On ARM-based systems (common in embedded Linux, servers, mobile), the **GIC** (Generic Interrupt Controller) is the equivalent of x86's APIC. The GICv3/v4 specification defines:

- **SGI (Software Generated Interrupts)**: Vectors 0–15, used for inter-processor communication (IPI).
- **PPI (Private Peripheral Interrupts)**: Vectors 16–31, per-CPU (e.g., the per-CPU timer).
- **SPI (Shared Peripheral Interrupts)**: Vectors 32–1019, shared peripheral devices.
- **LPI (Locality-specific Peripheral Interrupts)**: GICv3+, message-based (similar to MSI-X).

The GIC has a **Distributor** (global, one per system) that routes interrupts to **CPU Interfaces** (per CPU core). Each interrupt can be configured for:
- Priority (0–255, lower value = higher priority)
- Target CPU affinity
- Trigger type (level-sensitive vs. edge-triggered)

## 3. Interrupt Descriptor Table (IDT)

### 3.1 IDT Overview

The **IDT** (Interrupt Descriptor Table) is an array of 256 8-byte (in 32-bit) or 16-byte (in 64-bit) **gate descriptors** indexed by interrupt vector number. It is the kernel's "jump table" for interrupt and exception handling.

The CPU locates the IDT using the **IDTR** register (loaded with the `LIDT` instruction), which stores the base address and limit of the table.

### 3.2 Gate Descriptor Types

```
64-bit Gate Descriptor (16 bytes):
Bits [15:0]   — Offset bits 0-15 (handler address low)
Bits [31:16]  — Segment Selector (points to code segment in GDT)
Bits [34:32]  — IST (Interrupt Stack Table index, 0 = no switch)
Bits [39:35]  — Reserved (must be zero)
Bits [43:40]  — Type field:
                  1110b = 64-bit Interrupt Gate  (IF cleared on entry)
                  1111b = 64-bit Trap Gate       (IF preserved on entry)
Bit  [44]     — Must be zero (not a system segment)
Bits [46:45]  — DPL (Descriptor Privilege Level: 0=kernel, 3=user)
Bit  [47]     — P (Present bit, must be 1 for valid gate)
Bits [63:48]  — Offset bits 16-31 (handler address mid)
Bits [95:64]  — Offset bits 32-63 (handler address high)
Bits [127:96] — Reserved
```

**Interrupt Gate**: CPU clears `IF` flag on entry → interrupts are disabled in the handler.
**Trap Gate**: CPU preserves `IF` flag → interrupts remain enabled (used for system calls on x86-32).

### 3.3 IST (Interrupt Stack Table)

The IST is a mechanism in x86-64 to switch to a dedicated known-good stack when handling critical exceptions. This is essential for **Double Fault** (which can occur when the kernel stack overflows — using the overflowed stack would be fatal). Linux defines:

```c
/* arch/x86/include/asm/page_64_types.h */
#define EXCEPTION_STKSZ    PAGE_SIZE    /* 4KB per exception stack */

/* Stack indices in TSS.ist[] */
#define DOUBLEFAULT_STACK   1   /* Double Fault handler */
#define NMI_STACK           2   /* NMI handler */
#define DEBUG_STACK         3   /* Debug (#DB) handler */
#define MCE_STACK           4   /* Machine Check Exception */
```

### 3.4 Vector Number Assignments

```
Vector   | Use
---------|-----------------------------------------------------
0        | #DE — Divide Error
1        | #DB — Debug Exception
2        | NMI — Non-Maskable Interrupt
3        | #BP — Breakpoint (INT3)
4        | #OF — Overflow (INTO)
5        | #BR — Bound Range Exceeded (BOUND)
6        | #UD — Invalid Opcode
7        | #NM — Device Not Available (FPU not present/disabled)
8        | #DF — Double Fault (IST=1 in Linux)
9        | Coprocessor Segment Overrun (obsolete)
10       | #TS — Invalid TSS
11       | #NP — Segment Not Present
12       | #SS — Stack-Segment Fault
13       | #GP — General Protection Fault
14       | #PF — Page Fault (pushes CR2 = faulting address)
15       | Reserved
16       | #MF — x87 Floating-Point Exception
17       | #AC — Alignment Check
18       | #MC — Machine Check (IST=4 in Linux)
19       | #XM/#XF — SIMD Floating-Point Exception
20       | #VE — Virtualization Exception
21–31    | Reserved by Intel
32–127   | External Hardware IRQs (mapped by PIC/APIC)
128      | System Call (INT 0x80 on x86-32; SYSCALL on x86-64)
129–255  | Other software/hardware use
```

## 4. Interrupt Controllers: PIC, APIC, GIC

### 4.1 Legacy 8259A PIC

The original IBM PC used two cascaded Intel 8259A **Programmable Interrupt Controllers** (PICs) providing 15 usable IRQ lines (IRQ 0–7 on master, IRQ 8–15 on slave, with IRQ 2 used for cascade).

Default IRQ mappings:
```
IRQ 0  — Programmable Interval Timer (PIT) / System Timer
IRQ 1  — PS/2 Keyboard
IRQ 2  — Cascade (slave PIC)
IRQ 3  — COM2 / COM4 (Serial Port)
IRQ 4  — COM1 / COM3 (Serial Port)
IRQ 5  — LPT2 / Sound Card
IRQ 6  — Floppy Disk Controller
IRQ 7  — LPT1 / Parallel Port
IRQ 8  — Real-Time Clock (RTC)
IRQ 9  — ACPI / Available
IRQ 10 — Available
IRQ 11 — Available
IRQ 12 — PS/2 Mouse
IRQ 13 — FPU / Coprocessor
IRQ 14 — ATA Primary
IRQ 15 — ATA Secondary
```

The PIC maps its IRQ lines to CPU interrupt vectors. In the original PC BIOS, IRQs 0-7 were mapped to vectors 8-15 — which conflicts with CPU exception vectors! Linux remaps them to vectors 32–47 during initialization.

**EOI (End of Interrupt)**: After servicing an interrupt, the ISR must write to the PIC's EOI register (0x20 for master, 0xA0 for slave) to allow further interrupts. Linux's `handle_edge_irq` and `handle_level_irq` do this.

Modern systems ship with the PIC disabled (or absent) and use the APIC exclusively.

### 4.2 APIC (Advanced Programmable Interrupt Controller)

The APIC system consists of two components:

**Local APIC (LAPIC)**: One per CPU core, handles:
- Receiving interrupts from I/O APIC
- Inter-Processor Interrupts (IPIs) from other cores
- Per-CPU timer interrupts (LAPIC timer)
- Performance monitoring counter interrupts
- Thermal sensor interrupts
- Machine Check exceptions

**I/O APIC**: One (or more) per system, handles:
- Routing external device IRQ lines to one or more LAPICs
- Supports up to 24 IRQ inputs (standard) or 64+ (extended)
- Each entry in its **Redirection Table** specifies:
  - Destination LAPIC(s)
  - Delivery mode (Fixed, Lowest Priority, SMI, NMI, INIT, ExtINT)
  - Trigger mode (Edge / Level)
  - Polarity (Active High / Active Low)
  - Vector number

```
I/O APIC Redirection Table Entry (64-bit):
[7:0]   Vector         — CPU interrupt vector (32-255)
[10:8]  Delivery Mode  — 000=Fixed, 001=Lowest Prio, 100=NMI, 111=ExtINT
[11]    Dest Mode      — 0=Physical (APIC ID), 1=Logical (CPU set)
[12]    Delivery Status — 0=Idle, 1=Send Pending (read-only)
[13]    Polarity       — 0=Active High, 1=Active Low
[14]    Remote IRR     — Level-triggered IRR status (read-only)
[15]    Trigger Mode   — 0=Edge, 1=Level
[16]    Mask           — 1=Masked (interrupt disabled)
[63:56] Destination    — Physical APIC ID or Logical Destination
```

#### LAPIC Registers

The LAPIC is memory-mapped at physical address `0xFEE00000` (or relocated via MSR on x2APIC). Key registers:

```
Offset  Register Name             Description
0x020   LAPIC ID                  Unique CPU APIC identifier
0x030   LAPIC Version             
0x080   TPR (Task Priority)       Sets minimum interrupt priority accepted
0x090   APR (Arbitration Priority)
0x0A0   PPR (Processor Priority)  Current processor priority (max of TPR/ISRPR)
0x0B0   EOI                       Write 0 to signal End Of Interrupt
0x0D0   LDR (Logical Dest)        Logical destination for interrupt routing
0x0E0   DFR (Dest Format)         Flat or Cluster logical mode
0x0F0   Spurious Interrupt Vector  Also contains LAPIC Enable bit (bit 8)
0x100   ISR (In-Service Register)  Bitmask of interrupts currently being serviced
0x180   TMR (Trigger Mode)         Edge/Level for each vector
0x200   IRR (Interrupt Request)    Pending interrupts not yet dispatched
0x280   ESR (Error Status)         APIC error flags
0x300   ICR Low (Interrupt Cmd)    Send IPI — write triggers delivery
0x310   ICR High                   Destination APIC ID for IPI
0x320   LVT Timer                  Local vector table: timer
0x330   LVT Thermal                Local vector table: thermal sensor
0x340   LVT PMC                    Local vector table: perf monitor counter
0x350   LVT LINT0                  Local interrupt pin 0
0x360   LVT LINT1                  Local interrupt pin 1
0x370   LVT Error                  Error interrupt
0x380   Timer Initial Count        
0x390   Timer Current Count        
0x3E0   Timer Divide Config        
```

**EOI**: Critical — after processing a level-triggered interrupt, the ISR must write to the EOI register. For edge-triggered interrupts and IPIs, EOI is also required. Forgetting EOI will prevent all further interrupts at that priority level.

### 4.3 x2APIC

The x2APIC (introduced with Intel Nehalem) extends APIC addressing from 8-bit to 32-bit APIC IDs, supporting more than 255 CPUs. The interface changes from memory-mapped registers to MSRs (Model Specific Registers), accessed with `RDMSR`/`WRMSR`:

```
MSR base: 0x800
LAPIC ID:  MSR 0x802
EOI:       MSR 0x80B  (WRMSR with value 0)
ICR:       MSR 0x830  (64-bit, single write for IPI — no two-step!)
```

### 4.4 MSI (Message Signaled Interrupts)

MSI is a mechanism where a PCIe device signals an interrupt by **writing a specific value to a specific memory address** rather than asserting a dedicated wire. This eliminates the need for dedicated IRQ lines and allows a single device to have multiple interrupt vectors.

The LAPIC is programmed to recognize writes to `0xFEE00000 + offset` as interrupt requests. A PCIe device with MSI is configured with:
- **Message Address**: `0xFEE[dest_id][rh][dm]00000h`
- **Message Data**: Contains the vector number and delivery mode

MSI-X extends this to up to 2048 interrupt vectors per device, each with independent masking.

## 5. Linux Kernel IRQ Subsystem

### 5.1 Architecture Overview

Linux has a layered IRQ subsystem that abstracts hardware interrupt controllers behind a unified API:

```
User Space
────────────────────────────────────────────────────────────
Kernel Space

Driver Layer
  request_irq() / free_irq()
  ↕
Generic IRQ Layer (kernel/irq/)
  irq_desc[] — per-IRQ descriptor
  irq_chip   — hardware abstraction
  irq_domain — hw→linux IRQ mapping
  handle_irq_event() — calls action list
  ↕
IRQ Chip Layer (drivers/irqchip/)
  APIC, GIC, GPIO interrupt controllers
  irq_chip operations: mask, unmask, ack, eoi
  ↕
Hardware: PIC / APIC / GIC / GPIO
```

### 5.2 irq_desc: The Per-IRQ Descriptor

Every Linux IRQ number has an associated `struct irq_desc` — the central data structure of the IRQ subsystem:

```c
/* include/linux/irqdesc.h (simplified) */
struct irq_desc {
    struct irq_common_data   irq_common_data;  /* shared data */
    struct irq_data          irq_data;          /* per-irq chip data */
    unsigned int __percpu   *kstat_irqs;        /* per-CPU IRQ stats */
    irq_flow_handler_t       handle_irq;        /* high-level handler */
    struct irqaction        *action;            /* ISR action chain */
    unsigned int             status_use_accessors; /* IRQ status flags */
    unsigned int             core_internal_state__do_not_mess_with_it;
    unsigned int             depth;             /* nested disable depth */
    unsigned int             wake_depth;        /* wake on depth */
    unsigned int             tot_count;         /* total interrupt count */
    unsigned int             irq_count;         /* for detecting stuckness */
    unsigned long            last_unhandled;    /* timestamp for misrouting */
    unsigned int             irqs_unhandled;    /* count of spurious */
    atomic_t                 threads_handled;   
    int                      threads_handled_last;
    raw_spinlock_t           lock;              /* protects this structure */
    struct cpumask          *percpu_enabled;    
    struct cpumask           percpu_affinity;   
    const struct cpumask    *affinity_hint;     
    struct irq_affinity_notify *affinity_notify;
#ifdef CONFIG_IRQ_REMAP
    struct irq_2_iommu       irq_2_iommu;
#endif
    int                      parent_irq;        /* for hierarchical domains */
    struct module           *owner;             
    const char              *name;              /* /proc/interrupts name */
};
```

The `irq_desc` array is either a static array (if `CONFIG_SPARSE_IRQ` is not set) or a radix tree (for sparse allocation with `CONFIG_SPARSE_IRQ`).

### 5.3 struct irqaction: The Handler Chain

```c
/* include/linux/interrupt.h */
struct irqaction {
    irq_handler_t        handler;      /* The ISR function pointer */
    void                *dev_id;       /* Cookie passed to handler (device struct) */
    void __percpu       *percpu_dev_id;
    struct irqaction    *next;         /* Next action in shared IRQ chain */
    irq_handler_t        thread_fn;    /* Threaded IRQ handler function */
    struct task_struct  *thread;       /* Kernel thread for threaded IRQ */
    struct irqaction    *secondary;    /* Secondary actions */
    unsigned int         irq;          /* IRQ number */
    unsigned int         flags;        /* IRQF_* flags */
    unsigned long        thread_flags; /* IRQ thread flags */
    unsigned long        thread_mask;  /* Bitmask for thread sync */
    const char          *name;         /* /proc/interrupts name */
    struct proc_dir_entry *dir;        /* /proc/irq/N/name entry */
};
```

### 5.4 irq_chip: Hardware Abstraction

The `irq_chip` structure provides the interface between the generic IRQ layer and the specific interrupt controller hardware:

```c
/* include/linux/irq.h */
struct irq_chip {
    struct device   *parent_device;
    const char      *name;

    /* Flow control operations */
    void (*irq_enable)(struct irq_data *data);
    void (*irq_disable)(struct irq_data *data);
    void (*irq_ack)(struct irq_data *data);       /* Acknowledge (edge) */
    void (*irq_mask)(struct irq_data *data);      /* Mask (disable) IRQ */
    void (*irq_mask_ack)(struct irq_data *data);  /* Mask + Ack atomically */
    void (*irq_unmask)(struct irq_data *data);    /* Unmask (enable) IRQ */
    void (*irq_eoi)(struct irq_data *data);       /* End of Interrupt */

    /* Affinity */
    int  (*irq_set_affinity)(struct irq_data *data,
                              const struct cpumask *dest, bool force);
    int  (*irq_retrigger)(struct irq_data *data);

    /* Trigger type */
    int  (*irq_set_type)(struct irq_data *data, unsigned int flow_type);
    int  (*irq_set_wake)(struct irq_data *data, unsigned int on);

    /* Bus locking for slow busses (I2C, SPI) */
    void (*irq_bus_lock)(struct irq_data *data);
    void (*irq_bus_sync_unlock)(struct irq_data *data);

    /* CPU hotplug */
    void (*irq_cpu_online)(struct irq_data *data);
    void (*irq_cpu_offline)(struct irq_data *data);

    /* IRQ domain hierarchy */
    void (*irq_compose_msi_msg)(struct irq_data *data, struct msi_msg *msg);
    void (*irq_write_msi_msg)(struct irq_data *data, struct msi_msg *msg);

    /* Power management */
    int  (*irq_get_irqchip_state)(struct irq_data *data,
                                   enum irqchip_irq_state which, bool *state);
    int  (*irq_set_irqchip_state)(struct irq_data *data,
                                   enum irqchip_irq_state which, bool state);

    /* IRQ request/release */
    int  (*irq_request_resources)(struct irq_data *data);
    void (*irq_release_resources)(struct irq_data *data);

    unsigned long flags;
};
```

### 5.5 IRQ Domains

IRQ domains solve the problem of translating hardware interrupt numbers (as seen by the interrupt controller) to Linux virtual IRQ numbers (`virq`). This is essential because:
- A GPIO controller may have its own 0-based interrupt numbering.
- MSI-X vectors are allocated dynamically.
- Hierarchical interrupt controllers (GIC → GPIO) need to chain translations.

```c
/* include/linux/irqdomain.h */
struct irq_domain {
    struct list_head        link;       /* Linked list of all domains */
    const char             *name;       
    const struct irq_domain_ops *ops;  /* Translation operations */
    void                   *host_data; /* Private data for the domain */
    unsigned int            flags;      /* IRQ_DOMAIN_FLAG_* */
    unsigned int            mapcount;   /* Number of mapped interrupts */
    struct fwnode_handle   *fwnode;     /* Firmware node (DT/ACPI) */
    enum irq_domain_bus_token bus_token;
    struct irq_domain_chip_generic *gc; /* Generic chip data */
    struct irq_domain      *parent;     /* For hierarchy */
    /* ... */
    irq_hw_number_t         hwirq_max;  /* Max hardware IRQ */
    unsigned int            revmap_size;
    struct radix_tree_root  revmap_tree;/* hwirq → virq reverse map */
    struct mutex            revmap_mutex;
    struct irq_data __rcu  *revmap[];   /* Linear reverse map */
};

struct irq_domain_ops {
    int  (*match)(struct irq_domain *d, struct device_node *node,
                  enum irq_domain_bus_token bus_token);
    int  (*select)(struct irq_domain *d, struct irq_fwspec *fwspec,
                   enum irq_domain_bus_token bus_token);
    int  (*map)(struct irq_domain *d, unsigned int virq,
                irq_hw_number_t hw);
    void (*unmap)(struct irq_domain *d, unsigned int virq);
    int  (*xlate)(struct irq_domain *d, struct device_node *node,
                  const u32 *intspec, unsigned int intsize,
                  unsigned long *out_hwirq, unsigned int *out_type);
    /* Hierarchy operations */
    int  (*alloc)(struct irq_domain *d, unsigned int virq,
                  unsigned int nr_irqs, void *arg);
    void (*free)(struct irq_domain *d, unsigned int virq,
                 unsigned int nr_irqs);
    int  (*activate)(struct irq_domain *d, struct irq_data *irqd, bool reserve);
    void (*deactivate)(struct irq_domain *d, struct irq_data *irq_data);
    int  (*translate)(struct irq_domain *d, struct irq_fwspec *fwspec,
                      unsigned long *out_hwirq, unsigned int *out_type);
};
```

### 5.6 High-Level IRQ Flow Handlers

The `handle_irq` field in `irq_desc` points to a flow handler that implements the specific interrupt flow type:

```c
/* kernel/irq/chip.c */

/* Edge-triggered: ack immediately, then call handlers */
void handle_edge_irq(struct irq_desc *desc);

/* Level-triggered: mask, ack, call handlers, unmask */
void handle_level_irq(struct irq_desc *desc);

/* Fast EOI: for controllers that auto-mask on delivery */
void handle_fasteoi_irq(struct irq_desc *desc);

/* Per-CPU: for interrupts that go to one specific CPU only */
void handle_percpu_irq(struct irq_desc *desc);
void handle_percpu_devid_irq(struct irq_desc *desc);

/* Simple: no masking/unmasking (for always-on or managed elsewhere) */
void handle_simple_irq(struct irq_desc *desc);

/* Bad: for unconfigured/spurious IRQs */
void handle_bad_irq(struct irq_desc *desc);
```

**Edge vs. Level Triggered Explanation:**

*Edge-triggered*: Interrupt fires on a *transition* (low→high or high→low). The PIC/APIC does not re-assert the IRQ after acknowledgment. If the CPU misses an edge, the interrupt is lost. ISR must `ack` the interrupt controller before re-enabling — otherwise a second edge could be missed.

*Level-triggered*: Interrupt fires as long as the line is held in the active state. The controller keeps asserting the interrupt until the device is serviced (device clears the interrupt flag) AND the ISR unmasks the line. This is more reliable but requires the ISR to both service the device AND unmask the IRQ.

## 6. Writing ISRs in the Linux Kernel (C)

### 6.4 Minimal ISR Example

```c
#include <linux/interrupt.h>
#include <linux/module.h>
#include <linux/kernel.h>

#define MY_IRQ_NUMBER 11
#define DEVICE_NAME   "my_device"

static irqreturn_t my_isr(int irq, void *dev_id)
{
    /* dev_id is the cookie we passed to request_irq */
    struct my_device *dev = (struct my_device *)dev_id;

    /* Read the device's status register to determine if it caused this IRQ.
     * This is CRITICAL for shared IRQs — we must return IRQ_NONE if it
     * wasn't us, to allow the next handler in the chain to be called. */
    u32 status = ioread32(dev->base + STATUS_REG);
    if (!(status & IRQ_PENDING_BIT))
        return IRQ_NONE;  /* Not our interrupt */

    /* Clear the interrupt on the device (before acking to avoid races) */
    iowrite32(IRQ_CLEAR_BIT, dev->base + STATUS_REG);

    /* Do minimal work here — just record that an interrupt occurred
     * and wake up a bottom half to do the real work */
    dev->irq_count++;
    dev->last_status = status;

    /* Schedule bottom-half processing */
    tasklet_schedule(&dev->tasklet);

    return IRQ_HANDLED;
}

static int __init my_driver_init(void)
{
    int ret;
    
    /* ... initialize dev structure, map I/O memory ... */

    ret = request_irq(MY_IRQ_NUMBER,
                      my_isr,
                      IRQF_SHARED,    /* Shared IRQ line */
                      DEVICE_NAME,
                      dev);           /* dev_id — unique per device */
    if (ret) {
        pr_err("Failed to request IRQ %d: %d\n", MY_IRQ_NUMBER, ret);
        return ret;
    }
    return 0;
}

static void __exit my_driver_exit(void)
{
    /* free_irq waits for handler to complete before returning */
    free_irq(MY_IRQ_NUMBER, dev);
    /* ... clean up ... */
}
```

## 7. Top Half vs. Bottom Half Processing

### 7.1 The Problem with Long ISRs

An ISR runs in **interrupt context** with interrupts disabled (or at least the current IRQ line masked). This creates several constraints:

1. **No sleeping**: Cannot call `schedule()`, `msleep()`, `mutex_lock()`, or any function that may sleep.
2. **No user-space access**: Cannot use `copy_to_user()` or dereference user pointers.
3. **Limited stack**: ISRs run on a small, fixed kernel stack (typically 8KB or 16KB per CPU on x86).
4. **Interrupt latency**: Other interrupts are delayed while this ISR runs.

If a network card ISR were to copy gigabytes of data from the NIC ring buffer while in interrupt context, all other IRQs on that CPU would be blocked — catastrophic for system responsiveness.

### 7.2 The Top Half / Bottom Half Model

Linux splits interrupt handling into two phases:

**Top Half (Hardirq)**: The registered ISR. Runs immediately when the interrupt fires, in atomic (interrupt) context.
- Responsibilities: Acknowledge the interrupt to the hardware (so it can accept new interrupts), record minimal state (e.g., copy a few registers), schedule the bottom half.
- Rule: Must be **extremely fast** (microseconds).

**Bottom Half**: Deferred processing. Runs later, outside the critical interrupt context.
- Responsibilities: Process the data, allocate memory, wake up user processes.
- Can be interrupted by new top-half ISRs.
- Various mechanisms: **Softirqs**, **Tasklets**, **Workqueues**, **Threaded IRQs**.

```
Interrupt arrives
      │
      ▼
┌─────────────────────────────────┐
│      TOP HALF (ISR)             │
│  - Acknowledge hardware         │
│  - Read minimal status          │
│  - Copy critical data           │
│  - Schedule bottom half         │
│  - Return quickly (<10µs ideal) │
└─────────────┬───────────────────┘
              │ schedules
              ▼
┌─────────────────────────────────┐
│     BOTTOM HALF                 │
│  - Softirq / Tasklet / Workqueue│
│  - Process data fully           │
│  - Allocate/free memory         │
│  - Notify user space            │
│  - Can run in process context   │
└─────────────────────────────────┘
```

## 8. Softirqs

### 8.1 What Are Softirqs?

**Softirqs** (Software Interrupts) are the most performant bottom-half mechanism in Linux. They run with interrupts **enabled** but cannot sleep. Crucially, the **same softirq can run simultaneously on multiple CPUs** — they are designed for SMP scenarios and require careful concurrent programming.

Softirqs are statically allocated at compile time (you cannot create new ones in a module). There is a fixed set of 10 softirq types:

```c
/* include/linux/interrupt.h */
enum {
    HI_SOFTIRQ      = 0,  /* High-priority tasklets */
    TIMER_SOFTIRQ   = 1,  /* Timer callbacks (timer wheel) */
    NET_TX_SOFTIRQ  = 2,  /* Network: transmit packets */
    NET_RX_SOFTIRQ  = 3,  /* Network: receive packets (NAPI) */
    BLOCK_SOFTIRQ   = 4,  /* Block I/O completions */
    IRQ_POLL_SOFTIRQ = 5, /* IRQ poll (blk-mq polling) */
    TASKLET_SOFTIRQ = 6,  /* Normal-priority tasklets */
    SCHED_SOFTIRQ   = 7,  /* CFS/RT scheduler run queues */
    HRTIMER_SOFTIRQ = 8,  /* High-resolution timers */
    RCU_SOFTIRQ     = 9,  /* RCU callbacks (grace period end) */
    NR_SOFTIRQS     = 10
};
```

### 8.2 Softirq Execution

Softirqs are executed in several places:
1. **After each hardirq handler completes** (`irq_exit()` → `__do_softirq()`).
2. **After `local_bh_enable()`** (when bottom-halves are re-enabled).
3. **In `ksoftirqd` kernel threads** (one per CPU, runs when softirq load is high to prevent starvation).

```c
/* kernel/softirq.c — simplified __do_softirq() */
asmlinkage __visible void __softirq_entry __do_softirq(void)
{
    unsigned long end = jiffies + MAX_SOFTIRQ_TIME;  /* 2ms time budget */
    int max_restart = MAX_SOFTIRQ_RESTART;           /* 10 restarts max */
    struct softirq_action *h;
    u32 pending;

    pending = local_softirq_pending();
    
    /* account_irq_enter_time, etc. */
    __local_bh_disable_ip(_RET_IP_, SOFTIRQ_OFFSET);

restart:
    /* Reset pending, process each bit */
    set_softirq_pending(0);
    local_irq_enable();   /* Softirqs run with IRQs ENABLED */

    h = softirq_vec;
    while ((softirq_bit = ffs(pending))) {
        h += softirq_bit - 1;
        h->action(h);     /* Call the registered softirq handler */
        h++;
        pending >>= softirq_bit;
    }

    local_irq_disable();
    pending = local_softirq_pending();
    
    if (pending) {
        if (time_before(jiffies, end) && !need_resched() &&
            --max_restart)
            goto restart;
        /* Offload remaining work to ksoftirqd */
        wakeup_softirqd();
    }
    
    __local_bh_enable(SOFTIRQ_OFFSET);
}
```

### 8.3 Raising a Softirq

```c

}
```

## 9. Tasklets

### 9.1 Tasklets vs. Softirqs

**Tasklets** are built on top of softirqs (using `HI_SOFTIRQ` and `TASKLET_SOFTIRQ`). Key differences from softirqs:

| Property | Softirq | Tasklet |
|----------|---------|---------|
| Simultaneous execution on multiple CPUs | Yes | No (serialized) |
| Dynamic creation | No (compile-time only) | Yes (modules can create) |
| Concurrency model | Requires explicit locking | Guaranteed not to run concurrently |
| Performance | Highest | High |
| Use case | Network stack, timer, block | Most drivers |

A tasklet is guaranteed to **not run concurrently with itself** — it will only execute on one CPU at a time. However, different tasklets can run simultaneously on different CPUs.

### 9.2 struct tasklet_struct

```c
/* include/linux/interrupt.h */
struct tasklet_struct {
    struct tasklet_struct *next;  /* Linked list of pending tasklets */
    unsigned long          state; /* TASKLET_STATE_SCHED, TASKLET_STATE_RUN */
    atomic_t               count; /* Disable count (0 = enabled) */
    bool                   use_callback;
    union {
        void (*func)(unsigned long data);     /* Legacy callback */
        void (*callback)(struct tasklet_struct *t); /* New callback */
    };
    unsigned long          data;  /* Argument to func (legacy) */
};
```

### 9.3 Tasklet API


### 9.4 Complete Tasklet Example

```c
#include <linux/module.h>
#include <linux/interrupt.h>
#include <linux/kernel.h>

struct my_device {
    void __iomem    *base;
    int              irq;
    struct tasklet_struct tasklet;
    u32              pending_status;
    spinlock_t       lock;
};

/* Bottom half — runs in softirq context (no sleeping!) */
static void my_tasklet_handler(struct tasklet_struct *t)
{
    struct my_device *dev = from_tasklet(dev, t, tasklet);
    u32 status;
    unsigned long flags;

    spin_lock_irqsave(&dev->lock, flags);
    status = dev->pending_status;
    dev->pending_status = 0;
    spin_unlock_irqrestore(&dev->lock, flags);

    /* Process the captured status — still no sleeping! */
    if (status & RX_COMPLETE) {
        /* Read data from device ring buffer */
        process_rx_data(dev);
    }
    if (status & TX_COMPLETE) {
        complete_tx(dev);
    }
    
    pr_debug("Tasklet processed status: 0x%08x\n", status);
}

/* Top half ISR */
static irqreturn_t my_isr(int irq, void *dev_id)
{
    struct my_device *dev = dev_id;
    u32 status;

    status = ioread32(dev->base + IRQ_STATUS);
    if (!status)
        return IRQ_NONE;

    /* Acknowledge interrupt immediately */
    iowrite32(status, dev->base + IRQ_CLEAR);

    /* Save status and schedule bottom half */
    spin_lock(&dev->lock);
    dev->pending_status |= status;
    spin_unlock(&dev->lock);

    tasklet_schedule(&dev->tasklet);
    return IRQ_HANDLED;
}

static int my_probe(struct platform_device *pdev)
{
    struct my_device *dev;
    int ret;

    dev = devm_kzalloc(&pdev->dev, sizeof(*dev), GFP_KERNEL);
    if (!dev)
        return -ENOMEM;

    spin_lock_init(&dev->lock);
    tasklet_setup(&dev->tasklet, my_tasklet_handler);

    dev->irq = platform_get_irq(pdev, 0);
    ret = devm_request_irq(&pdev->dev, dev->irq, my_isr,
                            IRQF_SHARED, "my_device", dev);
    if (ret) {
        tasklet_kill(&dev->tasklet);
        return ret;
    }
    return 0;
}
```

> **Note (Linux 6.x):** Tasklets are soft-deprecated in newer kernels. The kernel developers recommend using threaded IRQs or workqueues instead for new drivers. Tasklets remain for existing code and specific performance-critical use cases.

## 10. Workqueues

### 10.1 Workqueues vs. Tasklets

**Workqueues** are the most flexible bottom-half mechanism. Unlike softirqs and tasklets, workqueue handlers run in **process context** — meaning they can sleep, allocate memory with `GFP_KERNEL`, call blocking functions, and use mutexes.

| Property | Softirq | Tasklet | Workqueue |
|----------|---------|---------|-----------|
| Context | Softirq (atomic) | Softirq (atomic) | Process (can sleep) |
| Can sleep | No | No | Yes |
| Can use mutex | No | No | Yes |
| Concurrency | Multi-CPU | Serialized per tasklet | Configurable |
| Latency | Lowest | Low | Higher |

### 10.2 The cmwq (Concurrency-Managed Workqueue) System

Linux uses **cmwq** (concurrency-managed workqueues) since kernel 2.6.36. cmwq uses a pool of worker threads managed by the system:

- `system_wq`: Default workqueue. Concurrency managed. Use `schedule_work()`.
- `system_highpri_wq`: High-priority, for time-sensitive work.
- `system_long_wq`: For long-running work items (doesn't tie up normal pool).
- `system_unbound_wq`: Not bound to specific CPUs.
- `system_freezable_wq`: Frozen during system suspend.
- `system_power_efficient_wq`: Prefers energy efficiency over performance.

### 10.5 Complete Workqueue Example

```c
#include <linux/module.h>
#include <linux/interrupt.h>
#include <linux/workqueue.h>

struct my_device {
    void __iomem           *base;
    int                     irq;
    struct work_struct      rx_work;
    struct delayed_work     timeout_work;
    struct workqueue_struct *wq;
    spinlock_t              lock;
    struct list_head        rx_queue;
};

/* Helper to get device from work_struct */
static inline struct my_device *work_to_dev(struct work_struct *work)
{
    return container_of(work, struct my_device, rx_work);
}

/* Workqueue handler — runs in process context, CAN sleep */
static void my_rx_work_handler(struct work_struct *work)
{
    struct my_device *dev = work_to_dev(work);
    struct rx_packet *pkt;
    unsigned long flags;
    
    /* Can use GFP_KERNEL allocations here */
    u8 *buffer = kmalloc(PAGE_SIZE, GFP_KERNEL);
    if (!buffer)
        return;

    spin_lock_irqsave(&dev->lock, flags);
    while (!list_empty(&dev->rx_queue)) {
        pkt = list_first_entry(&dev->rx_queue, struct rx_packet, list);
        list_del(&pkt->list);
        spin_unlock_irqrestore(&dev->lock, flags);

        /* Process packet — can sleep here */
        process_packet(dev, pkt);
        kfree(pkt);

        /* Can call blocking functions: */
        /* mutex_lock(&dev->mutex); */
        /* msleep(1); */
        /* copy_to_user(...); */

        spin_lock_irqsave(&dev->lock, flags);
    }
    spin_unlock_irqrestore(&dev->lock, flags);
    kfree(buffer);
}

static void my_timeout_handler(struct work_struct *work)
{
    struct delayed_work *dwork = to_delayed_work(work);
    struct my_device *dev = container_of(dwork, struct my_device, timeout_work);
    
    pr_warn("Device timeout — resetting\n");
    reset_device(dev);
    
    /* Reschedule itself for periodic checking */
    queue_delayed_work(dev->wq, &dev->timeout_work, HZ * 5);
}

/* ISR — top half only */
static irqreturn_t my_isr(int irq, void *dev_id)
{
    struct my_device *dev = dev_id;
    struct rx_packet *pkt;
    u32 status = ioread32(dev->base + STATUS_REG);
    
    if (!(status & IRQ_PENDING))
        return IRQ_NONE;

    iowrite32(IRQ_CLEAR, dev->base + STATUS_REG);

    if (status & RX_READY) {
        /* Allocate with GFP_ATOMIC — cannot sleep in ISR */
        pkt = kmalloc(sizeof(*pkt), GFP_ATOMIC);
        if (pkt) {
            /* Copy DMA descriptor reference — actual data copy in workqueue */
            pkt->dma_addr = ioread32(dev->base + RX_DMA_ADDR);
            pkt->length   = ioread32(dev->base + RX_LENGTH);
            
            spin_lock(&dev->lock);
            list_add_tail(&pkt->list, &dev->rx_queue);
            spin_unlock(&dev->lock);
            
            /* Schedule workqueue */
            queue_work(dev->wq, &dev->rx_work);
        }
    }
    return IRQ_HANDLED;
}

static int my_probe(struct platform_device *pdev)
{
    struct my_device *dev;
    
    dev = devm_kzalloc(&pdev->dev, sizeof(*dev), GFP_KERNEL);
    spin_lock_init(&dev->lock);
    INIT_LIST_HEAD(&dev->rx_queue);
    INIT_WORK(&dev->rx_work, my_rx_work_handler);
    INIT_DELAYED_WORK(&dev->timeout_work, my_timeout_handler);
    
    /* Create a private, ordered workqueue */
    dev->wq = alloc_workqueue("my_device_wq",
                               WQ_UNBOUND | WQ_MEM_RECLAIM,
                               1 /* max_active=1 for ordered execution */);
    if (!dev->wq)
        return -ENOMEM;

    /* Start periodic timeout check */
    queue_delayed_work(dev->wq, &dev->timeout_work, HZ * 5);
    
    return devm_request_irq(&pdev->dev, dev->irq, my_isr,
                             0, "my_device", dev);
}
```

## 11. Threaded IRQs

### 11.1 Concept

**Threaded IRQs** (introduced in Linux 2.6.30 and mandatory in `PREEMPT_RT`) move most interrupt processing into a dedicated kernel thread per IRQ. The top half ISR does the absolute minimum (check if it's our IRQ, acknowledge), then returns `IRQ_WAKE_THREAD` to wake the per-IRQ kernel thread.

Benefits:
- Thread can sleep, use mutexes, be assigned RT priority
- Cleaner code — no distinction between "can I sleep here?"
- `PREEMPT_RT` converts all IRQs to threaded by default

### 11.2 request_threaded_irq

```c
int request_threaded_irq(unsigned int irq,
                          irq_handler_t handler,       /* Top half (primary) */
                          irq_handler_t thread_fn,     /* Bottom half (thread) */
                          unsigned long flags,
                          const char *name,
                          void *dev_id);
```

If `handler` is NULL, Linux installs a default primary handler that just returns `IRQ_WAKE_THREAD` — useful when the controller auto-masks on delivery (e.g., GPIO IRQs with `IRQF_ONESHOT`).

`IRQF_ONESHOT`: The IRQ line remains **masked** until the threaded handler completes. This prevents a burst of interrupts before the thread can service the previous one.

### 11.3 Complete Threaded IRQ Example

```c
#include <linux/interrupt.h>
#include <linux/mutex.h>

struct my_i2c_device {
    struct i2c_client  *client;
    struct mutex        mutex;   /* Can use mutex in thread context */
    int                 irq;
    bool                data_ready;
    wait_queue_head_t   wait;
};

/* Top half — minimal, just acknowledge */
static irqreturn_t my_threaded_hard_handler(int irq, void *dev_id)
{
    struct my_i2c_device *dev = dev_id;
    
    /* For a GPIO interrupt, we often just let the thread handle everything.
     * Return IRQ_WAKE_THREAD to signal the threaded handler to run. */
    return IRQ_WAKE_THREAD;
}

/* Thread handler — runs in process context, CAN sleep */
static irqreturn_t my_threaded_handler(int irq, void *dev_id)
{
    struct my_i2c_device *dev = dev_id;
    u8 data[16];
    int ret;

    /* I2C read — sleeps while waiting for bus! This is fine in a thread. */
    mutex_lock(&dev->mutex);
    ret = i2c_smbus_read_i2c_block_data(dev->client, REG_DATA, 16, data);
    mutex_unlock(&dev->mutex);
    
    if (ret < 0) {
        dev_err(&dev->client->dev, "I2C read failed: %d\n", ret);
        return IRQ_HANDLED;
    }

    process_sensor_data(dev, data, ret);
    
    dev->data_ready = true;
    wake_up_interruptible(&dev->wait);  /* Wake up user-space reader */
    
    return IRQ_HANDLED;
}

static int my_i2c_probe(struct i2c_client *client)
{
    struct my_i2c_device *dev;
    int ret;
    
    dev = devm_kzalloc(&client->dev, sizeof(*dev), GFP_KERNEL);
    mutex_init(&dev->mutex);
    init_waitqueue_head(&dev->wait);
    dev->client = client;
    
    /* IRQF_ONESHOT: keep IRQ masked until threaded handler completes */
    ret = devm_request_threaded_irq(&client->dev,
                                     client->irq,
                                     my_threaded_hard_handler,  /* Primary */
                                     my_threaded_handler,        /* Thread */
                                     IRQF_TRIGGER_LOW | IRQF_ONESHOT,
                                     "my_i2c_sensor",
                                     dev);
    return ret;
}
```

## 12. IRQ Affinity and SMP

### 12.1 IRQ Balancing on SMP Systems

On multi-CPU systems, interrupt load can be distributed across CPUs to avoid any one CPU becoming a bottleneck. This is managed by:

1. **irqbalance daemon**: User-space daemon that monitors interrupt load via `/proc/interrupts` and adjusts IRQ affinity.
2. **Manual affinity**: Set via `/proc/irq/N/smp_affinity` or `smp_affinity_list`.
3. **Kernel IRQD_AFFINITY_MANAGED**: Driver/firmware-managed affinity (for NICs using RSS/RFS).

### 12.2 Per-CPU IRQs

Some interrupts are **per-CPU** — each CPU has its own instance (e.g., the LAPIC timer, performance monitoring counters). These are registered with `IRQF_PERCPU` and use per-CPU data structures:

```c
/* Register a per-CPU IRQ */
int request_percpu_irq(unsigned int irq,
                        irq_handler_t handler,
                        const char *devname,
                        void __percpu *percpu_dev_id);

/* Enable/disable on the current CPU */
void enable_percpu_irq(unsigned int irq, unsigned int type);
void disable_percpu_irq(unsigned int irq);
void free_percpu_irq(unsigned int irq, void __percpu *percpu_dev_id);
```

### 12.3 smp_affinity Files

```bash
# View IRQ affinity (bitmask: CPU 0 and 1 = 0x3)
cat /proc/irq/42/smp_affinity
00000003

# Set affinity to CPU 2 only
echo 4 > /proc/irq/42/smp_affinity

# Human-readable list format
cat /proc/irq/42/smp_affinity_list
0-1

echo "2" > /proc/irq/42/smp_affinity_list
```

### 12.4 Setting Affinity in Driver Code

```c
/* Set IRQ affinity hint (advisory, irqbalance may override) */
irq_set_affinity_hint(irq, cpumask_of(cpu));

/* Force affinity (hard, irqbalance cannot override) */
irq_set_affinity(irq_get_irq_data(irq), cpumask_of(cpu), true);

/* IRQ affinity notifier — callback when affinity changes */
struct irq_affinity_notify {
    unsigned int irq;
    struct kref kref;
    struct work_struct work;
    void (*notify)(struct irq_affinity_notify *, const cpumask_t *mask);
    void (*release)(struct kref *ref);
};

irq_set_affinity_notifier(irq, notify);
```

### 12.5 NUMA and IRQ Affinity

On NUMA systems, it's important to route IRQs to CPUs on the same NUMA node as the device's memory to minimize cross-node memory access:

```c
int node = dev_to_node(&pdev->dev);  /* Get device's NUMA node */
cpumask_t *mask = cpumask_of_node(node);
irq_set_affinity_hint(irq, mask);
```

## 13. Interrupt Context Rules and Constraints

### 13.1 What Is Interrupt Context?

Interrupt context means the CPU is not currently executing on behalf of any process. There is no associated `current` task (or `current` refers to whatever task happened to be running when the interrupt arrived — do not trust it in ISRs).

**What you CAN do in interrupt context:**
- Read/write device registers (`ioread8/16/32`, `iowrite8/16/32`)
- Spin locks (`spin_lock`, `spin_unlock`)
- Atomic operations (`atomic_add`, `atomic_cmpxchg`)
- `kmalloc` with `GFP_ATOMIC`
- `kfree`
- `schedule_work`, `tasklet_schedule`
- Wake up waiting processes (`wake_up`, `wake_up_interruptible`)
- Read/write per-CPU variables
- RCU read-side critical sections (`rcu_read_lock`, `rcu_dereference`)
- `printk` (but use sparingly due to performance impact)
- `mod_timer`, `del_timer`

**What you CANNOT do in interrupt context:**
- Sleep (`msleep`, `wait_event`, `schedule`)
- Block on a mutex (`mutex_lock`)
- Block on a semaphore (`down`)
- Allocate memory with `GFP_KERNEL` (may sleep)
- Access user space (`copy_to_user`, `copy_from_user`)
- Call `printk` with sync console (may deadlock)
- Disable preemption for long periods
- Take a sleeping lock held by an interrupt handler (deadlock!)

### 13.2 The GFP_ATOMIC Constraint

```c
/* In ISR — must use GFP_ATOMIC */
ptr = kmalloc(size, GFP_ATOMIC);  /* Non-blocking, may fail if memory is tight */

/* In process context — preferred */
ptr = kmalloc(size, GFP_KERNEL);  /* Can sleep to wait for memory */
```

`GFP_ATOMIC` allocates from a reserved emergency pool. It never sleeps. It can fail (`NULL` return) more readily than `GFP_KERNEL`. Always check the return value.

### 13.3 Detecting Interrupt Context in Code

```c
/* kernel/sched/core.c */
#define in_irq()              (hardirq_count())
#define in_softirq()          (softirq_count())
#define in_interrupt()        (irq_count())
#define in_serving_softirq()  (softirq_count() & SOFTIRQ_OFFSET)
#define in_nmi()              (preempt_count() & NMI_MASK)
#define in_task()             (!(in_nmi() | in_interrupt()))

/* Useful for assertions: */
might_sleep();          /* Warns (BUG in debug builds) if called in atomic context */
might_sleep_if(cond);   /* Conditional version */
cant_sleep();           /* Opposite — warns if NOT in atomic context */
```

### 13.4 Preemption Count

The kernel tracks "how atomic are we?" via `preempt_count()`, which is a per-thread counter (or per-CPU in interrupt context) broken into fields:

```
preempt_count bits:
[3:0]   PREEMPT_MASK  — Preemption disable nesting level
[11:4]  SOFTIRQ_MASK  — Softirq disable nesting level  
[19:12] HARDIRQ_MASK  — Hardirq (ISR) nesting level
[20]    NMI_MASK      — NMI context
```

- `spin_lock()` increments preemption count → `might_sleep()` warns
- `local_bh_disable()` increments softirq count
- Entering an ISR increments hardirq count
- NMI entry increments NMI count

## 14. Synchronization in ISR Context

### 14.1 The Problem

ISRs can interrupt *any* code, including code that holds data structure locks. This creates potential deadlocks and data races:

```c
/* Deadlock scenario: */
/* Process context: */
spin_lock(&dev->lock);        /* Acquires lock */
/* ... interrupt fires here ... */
    /* ISR context: */
    spin_lock(&dev->lock);    /* Tries to acquire same lock — DEADLOCK! */
/* ... ISR spins forever, process can never resume ... */
```

### 14.2 spin_lock_irqsave / spin_lock_irqrestore

The solution: disable local IRQs while holding locks that are also accessed from ISR context:

```c
unsigned long flags;

/* In process context — disable IRQs while holding lock */
spin_lock_irqsave(&dev->lock, flags);    /* Save RFLAGS, disable IRQs, acquire */
    /* ... critical section ... */
spin_unlock_irqrestore(&dev->lock, flags); /* Release, restore RFLAGS */

/* In ISR context — IRQs already disabled, but be safe: */
spin_lock(&dev->lock);    /* Just acquire (IRQs already off in ISR) */
    /* ... */
spin_unlock(&dev->lock);  /* Release */
```

**Rule**: If a lock is ever acquired in interrupt context, **all** acquisitions of that lock from process context must use `spin_lock_irqsave`.

### 14.3 spin_lock_bh: BH Locks

If a lock is shared between process context and softirq/tasklet context (but NOT hardirq), use `spin_lock_bh`:

```c
/* Disables softirqs (bottom halves) on local CPU while holding lock */
spin_lock_bh(&dev->lock);
    /* ... */
spin_unlock_bh(&dev->lock);
```

### 14.4 RCU in ISR Context

**RCU (Read-Copy-Update)** is safe in interrupt context for read-side operations:

```c
/* ISR reading an RCU-protected pointer */
static irqreturn_t my_isr(int irq, void *dev_id)
{
    struct config *cfg;
    
    rcu_read_lock();                     /* Forbids RCU grace periods */
    cfg = rcu_dereference(dev->config); /* Safely load pointer */
    if (cfg) {
        use_config(cfg);                 /* Safe while in RCU read lock */
    }
    rcu_read_unlock();
    
    return IRQ_HANDLED;
}

/* Process context updating the config */
static void update_config(struct my_device *dev, struct config *new_cfg)
{
    struct config *old_cfg;
    
    old_cfg = rcu_replace_pointer(dev->config, new_cfg, lockdep_is_held(&dev->lock));
    
    synchronize_rcu();  /* Wait for all RCU readers to finish */
    kfree(old_cfg);     /* Now safe to free */
}
```

### 14.5 Atomic Variables

For simple counters and flags shared between ISR and process context, `atomic_t` provides lock-free, interrupt-safe operations:

```c
atomic_t irq_count = ATOMIC_INIT(0);

/* ISR */
atomic_inc(&irq_count);

/* Process context */
int count = atomic_read(&irq_count);
atomic_set(&irq_count, 0);

/* Atomic operations */
atomic_add(int i, atomic_t *v);
atomic_sub(int i, atomic_t *v);
atomic_inc_return(atomic_t *v);   /* Returns new value */
atomic_dec_and_test(atomic_t *v); /* Returns true if result is zero */
atomic_cmpxchg(atomic_t *v, int old, int new); /* CAS */
```

### 14.6 Memory Barriers in ISR Context

Memory ordering between ISR and process context requires explicit barriers:

```c
/* ISR writes data, sets flag */
dev->data = read_from_hw();
smp_wmb();               /* Write memory barrier: data visible before flag */
dev->data_ready = 1;

/* Process context reads flag, then data */
if (dev->data_ready) {
    smp_rmb();           /* Read memory barrier: flag read before data */
    process(dev->data);
}

/* Paired write/read barrier */
smp_mb();                /* Full memory barrier (both read and write) */

/* For I/O memory (device registers) */
wmb();   /* Write barrier — all writes visible to device */
rmb();   /* Read barrier — read latest values from device */
mb();    /* Full I/O memory barrier */
```

## 15. Real-Time Linux (PREEMPT_RT)

### 15.1 The RT Patch and Mainline Integration

**PREEMPT_RT** (Real-Time Preemption) is a set of patches that transform Linux into a hard real-time OS. Key changes being progressively merged into mainline Linux (substantially complete as of 6.x):

1. **Threaded IRQs by default**: All interrupt handlers run as kernel threads, making them preemptible and schedulable.
2. **Sleeping spinlocks**: `spin_lock()` becomes a sleeping mutex under RT (called `rt_mutex`), eliminating priority inversion.
3. **High-resolution timers**: `hrtimer` infrastructure enables sub-millisecond timer precision.
4. **Preemptible kernel**: All kernel code paths are preemptible (except NMI and a few truly atomic sections).
5. **Priority inheritance**: RT mutexes support PI (Priority Inheritance) to resolve priority inversion.

### 15.2 Priority Inversion and PI

**Priority Inversion** is a classic real-time problem:

```
Scenario:
  Task H (high priority) — waiting for lock L held by Task L (low priority)
  Task M (medium priority) — running, preempting Task L
  Result: Task H is blocked by Task M even though H > M!

Solution: Priority Inheritance
  When Task H blocks on L, Task L inherits Task H's priority temporarily
  Task L runs at H's priority until it releases L
  Task H can then acquire L and run
```

Linux's `rt_mutex` (used internally by PREEMPT_RT for spinlocks) implements priority inheritance chains (PI chains).

### 15.3 ISR Configuration for RT

```c
/* Under PREEMPT_RT, force a specific IRQ to not be threaded (rare, dangerous): */

/* Set the RT thread priority for a threaded IRQ: */

```

## 16. ISR Latency, Timing, and Performance

### 16.1 Sources of Interrupt Latency

```
Total Interrupt Latency =
    Hardware propagation delay      (PCIe, wire, controller)
  + Interrupt controller processing (APIC, GIC routing)
  + CPU detection latency           (end of current instruction)
  + IRQ disabled period             (CLI region, spin locks)
  + ISR entry overhead              (save registers, switch stack)
  + ISR execution time              (your actual handler code)
  + EOI processing
  + Context restore overhead
```

### 16.2 Measuring ISR Latency

```c
/* Using kernel timestamps in the ISR */
static ktime_t irq_start_time;

static irqreturn_t latency_measuring_isr(int irq, void *dev_id)
{
    ktime_t now = ktime_get();
    ktime_t latency;
    
    /* Store the hardware event timestamp (if available from device) */
    u64 hw_timestamp = ioread64(dev->base + HW_TIMESTAMP);
    
    latency = ktime_sub(now, hw_timestamp_to_ktime(hw_timestamp));
    dev->latency_ns = ktime_to_ns(latency);
    
    /* Update statistics */
    if (dev->latency_ns > dev->max_latency_ns)
        dev->max_latency_ns = dev->latency_ns;
    
    return IRQ_HANDLED;
}
```

## 17. C Kernel Module: Complete ISR Examples

### 17.1 Complete GPIO IRQ Kernel Module

```c
// SPDX-License-Identifier: GPL-2.0
/*
 * gpio_irq_demo.c — Complete GPIO interrupt kernel module
 * Demonstrates: request_irq, threaded IRQ, workqueue, proc interface
 */

#include <linux/module.h>
#include <linux/kernel.h>
#include <linux/interrupt.h>
#include <linux/gpio.h>
#include <linux/gpio/consumer.h>
#include <linux/platform_device.h>
#include <linux/of.h>
#include <linux/of_gpio.h>
#include <linux/proc_fs.h>
#include <linux/seq_file.h>
#include <linux/workqueue.h>
#include <linux/ktime.h>
#include <linux/atomic.h>
#include <linux/spinlock.h>
#include <linux/slab.h>

MODULE_LICENSE("GPL");
MODULE_AUTHOR("Demo Author");
MODULE_DESCRIPTION("GPIO IRQ demonstration module");
MODULE_VERSION("1.0");

#define DRIVER_NAME    "gpio_irq_demo"
#define HISTORY_SIZE   16

struct irq_event {
    ktime_t  timestamp;
    u32      count;
};

struct gpio_irq_dev {
    struct gpio_desc       *gpio;
    int                     irq;
    
    /* Statistics — accessed from ISR, use atomics */
    atomic64_t              total_count;
    atomic64_t              missed_count;
    
    /* History ring buffer — protected by lock */
    spinlock_t              history_lock;
    struct irq_event        history[HISTORY_SIZE];
    int                     history_head;
    
    /* Workqueue for deferred processing */
    struct workqueue_struct *wq;
    struct work_struct       event_work;
    
    /* Proc filesystem entry */
    struct proc_dir_entry   *proc_entry;
    
    /* Pending events for workqueue */
    spinlock_t               pending_lock;
    u32                      pending_count;
    ktime_t                  pending_time;
};

static struct gpio_irq_dev *g_dev;

/* ======================== BOTTOM HALF: Workqueue ======================== */

static void gpio_event_work(struct work_struct *work)
{
    struct gpio_irq_dev *dev = container_of(work, struct gpio_irq_dev, event_work);
    struct irq_event event;
    unsigned long flags;
    
    /* Collect pending data */
    spin_lock_irqsave(&dev->pending_lock, flags);
    event.count     = dev->pending_count;
    event.timestamp = dev->pending_time;
    dev->pending_count = 0;
    spin_unlock_irqrestore(&dev->pending_lock, flags);
    
    if (!event.count)
        return;
    
    /* Store in history ring buffer */
    spin_lock_irqsave(&dev->history_lock, flags);
    dev->history[dev->history_head] = event;
    dev->history_head = (dev->history_head + 1) % HISTORY_SIZE;
    spin_unlock_irqrestore(&dev->history_lock, flags);
    
    pr_debug(DRIVER_NAME ": processed %u events at %lld ns\n",
             event.count, ktime_to_ns(event.timestamp));
}

/* ======================== TOP HALF: ISR ======================== */

static irqreturn_t gpio_irq_handler(int irq, void *dev_id)
{
    struct gpio_irq_dev *dev = dev_id;
    ktime_t now = ktime_get();

    /* Increment total count atomically (safe without lock) */
    atomic64_inc(&dev->total_count);

    /* Update pending data for workqueue */
    spin_lock(&dev->pending_lock);
    dev->pending_count++;
    dev->pending_time = now;
    spin_unlock(&dev->pending_lock);

    /* Schedule workqueue — idempotent, safe to call multiple times */
    queue_work(dev->wq, &dev->event_work);

    return IRQ_HANDLED;
}

/* ======================== PROC INTERFACE ======================== */

static int gpio_irq_proc_show(struct seq_file *m, void *v)
{
    struct gpio_irq_dev *dev = m->private;
    struct irq_event history[HISTORY_SIZE];
    int head;
    unsigned long flags;
    int i;

    /* Snapshot history */
    spin_lock_irqsave(&dev->history_lock, flags);
    memcpy(history, dev->history, sizeof(history));
    head = dev->history_head;
    spin_unlock_irqrestore(&dev->history_lock, flags);

    seq_printf(m, "GPIO IRQ Demo Statistics\n");
    seq_printf(m, "========================\n");
    seq_printf(m, "IRQ number:    %d\n", dev->irq);
    seq_printf(m, "Total events:  %lld\n", atomic64_read(&dev->total_count));
    seq_printf(m, "Missed:        %lld\n", atomic64_read(&dev->missed_count));
    seq_printf(m, "\nLast %d events:\n", HISTORY_SIZE);
    
    for (i = 0; i < HISTORY_SIZE; i++) {
        int idx = (head - 1 - i + HISTORY_SIZE) % HISTORY_SIZE;
        if (history[idx].count == 0)
            continue;
        seq_printf(m, "  [%2d] count=%u at %lld ns\n",
                   i, history[idx].count,
                   ktime_to_ns(history[idx].timestamp));
    }
    return 0;
}

static int gpio_irq_proc_open(struct inode *inode, struct file *file)
{
    return single_open(file, gpio_irq_proc_show, pde_data(inode));
}

static const struct proc_ops gpio_irq_proc_ops = {
    .proc_open    = gpio_irq_proc_open,
    .proc_read    = seq_read,
    .proc_lseek   = seq_lseek,
    .proc_release = single_release,
};

/* ======================== MODULE INIT/EXIT ======================== */

static int __init gpio_irq_demo_init(void)
{
    struct gpio_irq_dev *dev;
    int ret;

    dev = kzalloc(sizeof(*dev), GFP_KERNEL);
    if (!dev)
        return -ENOMEM;

    atomic64_set(&dev->total_count, 0);
    atomic64_set(&dev->missed_count, 0);
    spin_lock_init(&dev->history_lock);
    spin_lock_init(&dev->pending_lock);
    INIT_WORK(&dev->event_work, gpio_event_work);

    /* Request GPIO (platform-specific, normally from DT) */
    dev->gpio = gpio_to_desc(16);  /* GPIO 16 */
    if (IS_ERR(dev->gpio)) {
        ret = PTR_ERR(dev->gpio);
        pr_err(DRIVER_NAME ": Failed to get GPIO: %d\n", ret);
        goto err_gpio;
    }

    ret = gpiod_direction_input(dev->gpio);
    if (ret) {
        pr_err(DRIVER_NAME ": GPIO direction failed: %d\n", ret);
        goto err_gpio;
    }

    dev->irq = gpiod_to_irq(dev->gpio);
    if (dev->irq < 0) {
        ret = dev->irq;
        pr_err(DRIVER_NAME ": GPIO to IRQ failed: %d\n", ret);
        goto err_gpio;
    }

    /* Create workqueue */
    dev->wq = alloc_workqueue(DRIVER_NAME "_wq",
                               WQ_UNBOUND | WQ_MEM_RECLAIM,
                               0);
    if (!dev->wq) {
        ret = -ENOMEM;
        goto err_wq;
    }

    /* Register ISR — rising and falling edges */
    ret = request_irq(dev->irq,
                      gpio_irq_handler,
                      IRQF_TRIGGER_RISING | IRQF_TRIGGER_FALLING,
                      DRIVER_NAME,
                      dev);
    if (ret) {
        pr_err(DRIVER_NAME ": request_irq failed: %d\n", ret);
        goto err_irq;
    }

    /* Create /proc entry */
    dev->proc_entry = proc_create_data(DRIVER_NAME, 0444, NULL,
                                        &gpio_irq_proc_ops, dev);
    if (!dev->proc_entry) {
        pr_warn(DRIVER_NAME ": Failed to create /proc entry\n");
        /* Non-fatal */
    }

    g_dev = dev;
    pr_info(DRIVER_NAME ": Initialized, IRQ=%d\n", dev->irq);
    return 0;

err_irq:
    destroy_workqueue(dev->wq);
err_wq:
err_gpio:
    kfree(dev);
    return ret;
}

static void __exit gpio_irq_demo_exit(void)
{
    struct gpio_irq_dev *dev = g_dev;

    if (dev->proc_entry)
        proc_remove(dev->proc_entry);

    /* Disable and wait for ISR to finish */
    free_irq(dev->irq, dev);

    /* Flush and destroy workqueue */
    flush_workqueue(dev->wq);
    destroy_workqueue(dev->wq);

    kfree(dev);
    pr_info(DRIVER_NAME ": Unloaded\n");
}

module_exit(gpio_irq_demo_exit);
```

## 18. Go: ISR-Adjacent Programming

### 18.1 Go's Relationship with ISRs

Go cannot write Linux kernel ISRs directly — kernel modules must be written in C (or Rust with the in-kernel Rust support). However, Go is commonly used to:

1. **Write user-space drivers** using UIO (Userspace I/O) or VFIO.
2. **Interact with `/dev/uio*`** to handle hardware interrupts in user space.
3. **Use signal handlers** (`SIGINT`, `SIGIO`) as a form of software interrupt handling.
4. **Implement interrupt-like patterns** using goroutines and channels.

### 18.2 UIO (Userspace I/O) Interrupt Handling in Go

UIO allows device drivers to run in user space. Interrupts are received by reading from `/dev/uioN`:

```go
// uio_interrupt.go — Handle hardware interrupts via UIO from Go

import (
    "context"

// UIODevice represents a UIO device with interrupt handling
type UIODevice struct {
    fd          int
    name        string
    irqCount    atomic.Uint64
    missedCount atomic.Uint64

    // Channel to deliver interrupt events
    events chan IRQEvent

    // Context for lifecycle management
    ctx    context.Context
    cancel context.CancelFunc
    wg     sync.WaitGroup
}

// IRQEvent carries information about a received interrupt
type IRQEvent struct {
    Timestamp  time.Time
    Count      uint32
    SeqNumber  uint64
}

// NewUIODevice opens a UIO device and starts the interrupt listener.
// path: e.g., "/dev/uio0"
func NewUIODevice(path string) (*UIODevice, error) {
    fd, err := unix.Open(path, unix.O_RDWR, 0)
    if err != nil {
        return nil, fmt.Errorf("open %s: %w", path, err)
    }

    ctx, cancel := context.WithCancel(context.Background())
    dev := &UIODevice{
        fd:     fd,
        name:   path,
        events: make(chan IRQEvent, 256), // Buffered to avoid dropping
        ctx:    ctx,
        cancel: cancel,
    }

    dev.wg.Add(1)
    go dev.irqLoop()

    return dev, nil
}

// irqLoop is the goroutine that blocks on UIO reads (interrupt waits)
func (d *UIODevice) irqLoop() {
    defer d.wg.Done()
    defer close(d.events)

    // Pin this goroutine to an OS thread for deterministic scheduling
    runtime.LockOSThread()
    defer runtime.UnlockOSThread()

    var seq uint64
    buf := make([]byte, 4)

    for {
        // Check context cancellation using non-blocking poll first
        select {
        case <-d.ctx.Done():
            return
        default:
        }

        // Use poll() with timeout so we can check context periodically
        fds := []unix.PollFd{
            {Fd: int32(d.fd), Events: unix.POLLIN},
        }
        n, err := unix.Poll(fds, 100) // 100ms timeout
        if err != nil {
            if err == unix.EINTR {
                continue // Interrupted by signal, retry
            }
            log.Printf("poll error: %v", err)
            return
        }
        if n == 0 {
            continue // Timeout — check context and retry
        }
        if fds[0].Revents&unix.POLLIN == 0 {
            continue
        }

        // Reading from /dev/uioN blocks until an interrupt occurs.
        // Returns the interrupt count (4-byte little-endian uint32).
        ts := time.Now()
        _, err = unix.Read(d.fd, buf)
        if err != nil {
            log.Printf("read UIO: %v", err)
            return
        }

        irqCount := binary.LittleEndian.Uint32(buf)
        seq++

        d.irqCount.Add(uint64(irqCount))

        // Re-enable the interrupt (write 1 to UIO fd)
        // This unmasks the interrupt in the kernel UIO driver
        enableBuf := []byte{1, 0, 0, 0}
        unix.Write(d.fd, enableBuf)

        // Deliver event — non-blocking send
        event := IRQEvent{
            Timestamp: ts,
            Count:     irqCount,
            SeqNumber: seq,
        }
        select {
        case d.events <- event:
        default:
            // Channel full — increment missed count
            d.missedCount.Add(1)
        }
    }
}

// Events returns the channel of interrupt events
func (d *UIODevice) Events() <-chan IRQEvent {
    return d.events
}

// Close shuts down the UIO device listener
func (d *UIODevice) Close() error {
    d.cancel()
    d.wg.Wait()
    return unix.Close(d.fd)
}

// Stats returns interrupt statistics
func (d *UIODevice) Stats() (total, missed uint64) {
    return d.irqCount.Load(), d.missedCount.Load()
}

// ============ mmap for register access ============

// MapRegisters maps UIO device registers into process address space
func (d *UIODevice) MapRegisters(mapIndex int, size uintptr) ([]byte, error) {
    // UIO maps are at offset: map_index * page_size
    pageSize := uintptr(os.Getpagesize())
    offset := uintptr(mapIndex) * pageSize

    data, err := unix.Mmap(d.fd, int64(offset), int(size),
        unix.PROT_READ|unix.PROT_WRITE, unix.MAP_SHARED)
    if err != nil {
        return nil, fmt.Errorf("mmap UIO map %d: %w", mapIndex, err)
    }
    return data, nil
}

// ReadReg32 reads a 32-bit device register from mapped memory
func ReadReg32(mem []byte, offset uint32) uint32 {
    ptr := (*uint32)(unsafe.Pointer(&mem[offset]))
    // Volatile-like read (prevents compiler optimization)
    return atomic.LoadUint32(ptr)
}

// WriteReg32 writes a 32-bit device register
func WriteReg32(mem []byte, offset, value uint32) {
    ptr := (*uint32)(unsafe.Pointer(&mem[offset]))
    atomic.StoreUint32(ptr, value)
}
```

### 18.3 Go: Interrupt-Safe Ring Buffer

A common pattern in ISR programming is a lock-free ring buffer between producer (ISR) and consumer (process context):

```go
// ring_buffer.go — Lock-free single-producer single-consumer ring buffer
// Suitable for ISR-to-process communication patterns
package ringbuf

import (
    "runtime"
    "sync/atomic"
    "unsafe"
)

// PowerOf2 size ring buffer for SPSC use (ISR produces, process consumes)
type RingBuffer[T any] struct {
    _    [64]byte // Padding to prevent false sharing
    head atomic.Uint64 // Written by producer (ISR equivalent)
    _    [56]byte
    tail atomic.Uint64 // Written by consumer
    _    [56]byte
    mask uint64
    data []T
}

// NewRingBuffer creates a ring buffer with capacity rounded up to power of 2
func NewRingBuffer[T any](capacity uint64) *RingBuffer[T] {
    // Round up to next power of 2
    size := uint64(1)
    for size < capacity {
        size <<= 1
    }
    return &RingBuffer[T]{
        mask: size - 1,
        data: make([]T, size),
    }
}

// TryPush attempts to add an item. Returns false if full.
// Must be called from single producer only.
func (r *RingBuffer[T]) TryPush(item T) bool {
    head := r.head.Load()
    tail := r.tail.Load()
    
    // Check if full
    if head-tail > r.mask {
        return false
    }
    
    // Write item
    r.data[head&r.mask] = item
    
    // Memory barrier: ensure item is written before head is advanced
    atomic.StoreUint64((*uint64)(unsafe.Pointer(&r.head)), head+1)
    return true
}

// TryPop attempts to read an item. Returns false if empty.
// Must be called from single consumer only.
func (r *RingBuffer[T]) TryPop() (T, bool) {
    tail := r.tail.Load()
    head := r.head.Load()
    
    if tail == head {
        var zero T
        return zero, false
    }
    
    // Read item
    item := r.data[tail&r.mask]
    
    // Memory barrier before advancing tail
    atomic.StoreUint64((*uint64)(unsafe.Pointer(&r.tail)), tail+1)
    return item, true
}

// Drain calls fn for each available item
func (r *RingBuffer[T]) Drain(fn func(T)) int {
    count := 0
    for {
        item, ok := r.TryPop()
        if !ok {
            break
        }
        fn(item)
        count++
        runtime.Gosched() // Yield to allow other goroutines
    }
    return count
}

// Size returns approximate number of items
func (r *RingBuffer[T]) Size() uint64 {
    head := r.head.Load()
    tail := r.tail.Load()
    return head - tail
}
```

### 19.3 Rust Embedded ISR (no_std, bare metal)

For embedded systems (Cortex-M, RISC-V), Rust is the dominant systems language for new projects. The RTIC (Real-Time Interrupt-driven Concurrency) framework is the standard:

```rust
//! Complete RTIC application with ISR handling for Cortex-M
//! Cargo.toml dependencies:
//!   cortex-m = "0.7"
//!   cortex-m-rt = "0.7"
//!   rtic = { version = "2.0", features = ["thumbv7-backend"] }
//!   stm32f4xx-hal = { version = "0.20", features = ["stm32f411"] }
//!   heapless = "0.8"

#![no_std]
#![no_main]

use panic_halt as _;

#[rtic::app(device = stm32f4xx_hal::pac, dispatchers = [SPI1, SPI2])]
mod app {
    use stm32f4xx_hal::{
        gpio::{Edge, Input, PA0},
        pac,
        prelude::*,
        timer::{CounterUs, Event, Timer},
    };
    use heapless::spsc::{Consumer, Producer, Queue};
    use cortex_m::peripheral::NVIC;

    // Event types passed between ISR and task
    #[derive(Debug, Clone, Copy)]
    pub enum IrqEvent {
        ButtonPress { timestamp_us: u32 },
        TimerTick    { count: u32 },
        Error        { code: u8 },
    }

    // Shared resources — accessible from multiple tasks/ISRs
    #[shared]
    struct Shared {
        irq_count: u32,
        #[lock_free] // Atomic access — safe for single-priority access
        event_count: u32,
    }

    // Local resources — exclusive to a specific task/ISR
    #[local]
    struct Local {
        button: PA0<Input>,
        timer: CounterUs<pac::TIM2>,
        producer: Producer<'static, IrqEvent, 32>,
        consumer: Consumer<'static, IrqEvent, 32>,
    }

    // Static queue in BSS — zero-cost, no heap
    static mut EVENT_QUEUE: Queue<IrqEvent, 32> = Queue::new();

    #[init]
    fn init(ctx: init::Context) -> (Shared, Local) {
        let rcc = ctx.device.RCC.constrain();
        let clocks = rcc.cfgr.sysclk(84.MHz()).freeze();

        // Configure GPIO button with interrupt
        let gpioa = ctx.device.GPIOA.split();
        let mut button = gpioa.pa0.into_pull_down_input();
        
        // Configure the button to generate an interrupt on rising edge
        let mut syscfg = ctx.device.SYSCFG.constrain();
        button.make_interrupt_source(&mut syscfg);
        button.trigger_on_edge(&ctx.device.EXTI, Edge::Rising);
        button.enable_interrupt(&ctx.device.EXTI);

        // Configure timer for periodic interrupts at 1kHz
        let mut timer = ctx.device.TIM2.counter_us(&clocks);
        timer.start(1_000.micros()).unwrap(); // 1ms period
        timer.listen(Event::Update);

        // Split the queue into producer (for ISRs) and consumer (for tasks)
        let (producer, consumer) = unsafe { EVENT_QUEUE.split() };

        (
            Shared {
                irq_count: 0,
                event_count: 0,
            },
            Local {
                button,
                timer,
                producer,
                consumer,
            },
        )
    }

    // ============ ISR: EXTI0 — Button press (top half) ============
    // Priority 4 (higher = more urgent in RTIC's inversion)
    #[task(binds = EXTI0, local = [button, producer], shared = [irq_count], priority = 4)]
    fn button_isr(ctx: button_isr::Context) {
        // Clear the interrupt pending bit on the EXTI peripheral
        ctx.local.button.clear_interrupt_pending_bit();

        // Read timestamp from DWT cycle counter (Cortex-M debug unit)
        let timestamp_us = cortex_m::peripheral::DWT::cycle_count() / 84; // 84 MHz clock

        // Increment shared counter — uses RTIC's lock mechanism
        ctx.shared.irq_count.lock(|count| {
            *count += 1;
        });

        // Enqueue event for bottom-half processing — never blocks (SPSC)
        let event = IrqEvent::ButtonPress { timestamp_us };
        if ctx.local.producer.enqueue(event).is_err() {
            // Queue full — could set an error flag
        }

        // Spawn the processing task at lower priority (bottom half)
        process_events::spawn().ok(); // ok() ignores "already spawned" error
    }

    // ============ ISR: TIM2 — Timer tick ============
    #[task(binds = TIM2, local = [timer, producer], shared = [irq_count], priority = 3)]
    fn timer_isr(ctx: timer_isr::Context) {
        // Clear the timer update interrupt flag
        ctx.local.timer.clear_interrupt(Event::Update);

        ctx.shared.irq_count.lock(|count| {
            let c = *count;
            let event = IrqEvent::TimerTick { count: c };
            ctx.local.producer.enqueue(event).ok();
        });
    }

    // ============ Bottom half: async task (process context equivalent) ============
    // Priority 2 — lower than ISRs, can be preempted by them
    #[task(local = [consumer], shared = [event_count], priority = 2)]
    async fn process_events(ctx: process_events::Context) {
        // Drain the event queue
        while let Some(event) = ctx.local.consumer.dequeue() {
            ctx.shared.event_count.lock(|count| {
                *count += 1;
            });

            match event {
                IrqEvent::ButtonPress { timestamp_us } => {
                    // Handle button press — could debounce here
                    handle_button(timestamp_us);
                }
                IrqEvent::TimerTick { count } => {
                    handle_tick(count);
                }
                IrqEvent::Error { code } => {
                    handle_error(code);
                }
            }
        }
    }

    // ============ Idle task ============
    #[idle]
    fn idle(_: idle::Context) -> ! {
        loop {
            // WFI: Wait For Interrupt — enter low-power state until next IRQ
            cortex_m::asm::wfi();
        }
    }

    // Helper functions (would be in proper modules)
    fn handle_button(timestamp_us: u32) {
        let _ = timestamp_us;
        // Implement button logic
    }

    fn handle_tick(count: u32) {
        let _ = count;
        // Implement timer logic
    }

    fn handle_error(code: u8) {
        let _ = code;
        // Implement error handling
    }
}
```

### 19.4 Rust: Safe Spinlock for ISR Context

```rust
// safe_irq_spinlock.rs — ISR-safe spinlock abstraction in Rust
// Demonstrates how to implement interrupt-safe locking in no_std Rust

#![no_std]

use core::{
    cell::UnsafeCell,
    ops::{Deref, DerefMut},
    sync::atomic::{AtomicBool, Ordering},
};

/// A spinlock that also saves/restores interrupt state.
/// The SAVE type parameter is the architecture-specific interrupt state type.
pub struct IrqSpinLock<T> {
    locked: AtomicBool,
    data:   UnsafeCell<T>,
}

/// Guard that restores interrupt state on drop
pub struct IrqSpinLockGuard<'a, T> {
    lock:           &'a IrqSpinLock<T>,
    saved_primask:  u32, // Cortex-M PRIMASK register
}

unsafe impl<T: Send> Send for IrqSpinLock<T> {}
unsafe impl<T: Send> Sync for IrqSpinLock<T> {}

impl<T> IrqSpinLock<T> {
    pub const fn new(data: T) -> Self {
        IrqSpinLock {
            locked: AtomicBool::new(false),
            data:   UnsafeCell::new(data),
        }
    }

    /// Lock the spinlock, disabling interrupts.
    /// Returns a guard that re-enables interrupts on drop.
    pub fn lock(&self) -> IrqSpinLockGuard<'_, T> {
        // Save current interrupt state and disable interrupts
        let saved_primask = unsafe { cortex_m_disable_irq() };

        // Spin until we acquire the lock
        while self.locked.compare_exchange_weak(
            false, true,
            Ordering::Acquire,
            Ordering::Relaxed,
        ).is_err() {
            core::hint::spin_loop();
        }

        IrqSpinLockGuard {
            lock: self,
            saved_primask,
        }
    }
}

impl<'a, T> Deref for IrqSpinLockGuard<'a, T> {
    type Target = T;
    fn deref(&self) -> &T {
        unsafe { &*self.lock.data.get() }
    }
}

impl<'a, T> DerefMut for IrqSpinLockGuard<'a, T> {
    fn deref_mut(&mut self) -> &mut T {
        unsafe { &mut *self.lock.data.get() }
    }
}

impl<'a, T> Drop for IrqSpinLockGuard<'a, T> {
    fn drop(&mut self) {
        // Release the lock
        self.lock.locked.store(false, Ordering::Release);
        // Restore interrupt state
        unsafe { cortex_m_restore_irq(self.saved_primask) };
    }
}

/// Save PRIMASK and disable all maskable interrupts (CPSID i)
unsafe fn cortex_m_disable_irq() -> u32 {
    let primask: u32;
    core::arch::asm!(
        "mrs {0}, primask",  // Read current PRIMASK into register
        "cpsid i",           // Disable interrupts (set PRIMASK.PM)
        out(reg) primask,
        options(nomem, nostack, preserves_flags)
    );
    primask
}

/// Restore PRIMASK (re-enables interrupts if they were enabled before)
unsafe fn cortex_m_restore_irq(primask: u32) {
    core::arch::asm!(
        "msr primask, {0}",  // Restore PRIMASK
        in(reg) primask,
        options(nomem, nostack, preserves_flags)
    );
}
```

### 19.5 Rust: Lock-Free ISR Ring Buffer

```rust
// lock_free_ring.rs — Lock-free ring buffer for ISR-to-task communication
// Uses atomics only — safe for ISR producer, task consumer

use core::sync::atomic::{AtomicUsize, Ordering};
use core::cell::UnsafeCell;
use core::mem::MaybeUninit;

/// Single-Producer, Single-Consumer ring buffer
/// Producer (ISR) and Consumer (task) can operate concurrently without locks.
pub struct SpscRing<T, const N: usize> {
    head:   AtomicUsize,                     // Written by producer
    tail:   AtomicUsize,                     // Written by consumer  
    buffer: UnsafeCell<[MaybeUninit<T>; N]>,
}

// Safety: T must be Send for cross-thread/ISR access
unsafe impl<T: Send, const N: usize> Sync for SpscRing<T, N> {}
unsafe impl<T: Send, const N: usize> Send for SpscRing<T, N> {}

impl<T: Copy, const N: usize> SpscRing<T, N> {
    pub const fn new() -> Self {
        assert!(N.is_power_of_two(), "Capacity must be power of 2");
        SpscRing {
            head:   AtomicUsize::new(0),
            tail:   AtomicUsize::new(0),
            buffer: UnsafeCell::new(unsafe { MaybeUninit::uninit().assume_init() }),
        }
    }

    const MASK: usize = N - 1;

    /// Try to push an item. Returns Err(item) if full.
    /// Must only be called from the producer (ISR).
    pub fn try_push(&self, item: T) -> Result<(), T> {
        let head = self.head.load(Ordering::Relaxed);
        let tail = self.tail.load(Ordering::Acquire);

        if head.wrapping_sub(tail) >= N {
            return Err(item); // Buffer full
        }

        // Write item to slot
        unsafe {
            let slot = &mut (*self.buffer.get())[head & Self::MASK];
            slot.write(item);
        }

        // Release barrier: item must be written before head advances
        self.head.store(head.wrapping_add(1), Ordering::Release);
        Ok(())
    }

    /// Try to pop an item. Returns None if empty.
    /// Must only be called from the consumer (task).
    pub fn try_pop(&self) -> Option<T> {
        let tail = self.tail.load(Ordering::Relaxed);
        let head = self.head.load(Ordering::Acquire);

        if tail == head {
            return None; // Buffer empty
        }

        // Read item from slot
        let item = unsafe {
            (*self.buffer.get())[tail & Self::MASK].assume_init_read()
        };

        // Release barrier: item must be read before tail advances
        self.tail.store(tail.wrapping_add(1), Ordering::Release);
        Some(item)
    }

    /// Returns the number of items available
    pub fn len(&self) -> usize {
        let head = self.head.load(Ordering::Relaxed);
        let tail = self.tail.load(Ordering::Relaxed);
        head.wrapping_sub(tail)
    }

    pub fn is_empty(&self) -> bool { self.len() == 0 }
    pub fn is_full(&self) -> bool { self.len() >= N }
}

// ============ Usage ============

#[derive(Copy, Clone, Debug)]

    timestamp: u32,
    value: i16,
    channel: u8,
}

// Placed in static memory (no heap needed)

/// Simulated sensor ISR

        timestamp,
        value: raw_value as i16,

    };

        // Handle overflow — could set a flag, blink an LED, etc.

    }
}
```

## 21. Common Pitfalls and Anti-Patterns

### 21.1 Sleeping in Interrupt Context

```c
/* WRONG — Will cause BUG: scheduling while atomic */
static irqreturn_t bad_isr(int irq, void *dev_id)
{
    u8 *buf = kmalloc(64, GFP_KERNEL);  /* GFP_KERNEL may sleep — BUG! */
    msleep(10);                          /* sleeping — BUG! */
    mutex_lock(&dev->mutex);            /* may sleep — BUG! */
    return IRQ_HANDLED;
}

/* CORRECT */
static irqreturn_t good_isr(int irq, void *dev_id)
{
    u8 *buf = kmalloc(64, GFP_ATOMIC);  /* GFP_ATOMIC never sleeps */
    if (!buf) { /* handle allocation failure */ }
    schedule_work(&dev->work);           /* defer mutex/sleep work */
    return IRQ_HANDLED;
}
```

### 21.2 Forgetting IRQ-Safe Locking

```c
/* WRONG — Can deadlock if ISR fires while lock held in process context */
static void process_context_function(struct my_device *dev)
{
    spin_lock(&dev->lock);      /* Locks without disabling IRQs */
    /* ... if ISR fires here and tries to take dev->lock: DEADLOCK */
    spin_unlock(&dev->lock);
}

/* CORRECT */
static void process_context_function(struct my_device *dev)
{
    unsigned long flags;
    spin_lock_irqsave(&dev->lock, flags);   /* Disables IRQs */
    /* ISR cannot interrupt us here */
    spin_unlock_irqrestore(&dev->lock, flags);
}
```

### 21.3 Not Returning IRQ_NONE for Shared IRQs

```c
/* WRONG — Always returns IRQ_HANDLED even if not our interrupt */
static irqreturn_t wrong_shared_isr(int irq, void *dev_id)
{
    /* Does work regardless of whether we caused the interrupt */
    do_work(dev_id);
    return IRQ_HANDLED;  /* Incorrect for shared IRQ! */
}

/* CORRECT — Check if we caused the interrupt */
static irqreturn_t correct_shared_isr(int irq, void *dev_id)
{
    struct my_device *dev = dev_id;
    u32 status = ioread32(dev->base + STATUS_REG);
    
    if (!(status & MY_IRQ_BIT))
        return IRQ_NONE;  /* Not our interrupt */
    
    iowrite32(MY_IRQ_BIT, dev->base + STATUS_REG); /* Clear */
    do_work(dev);
    return IRQ_HANDLED;
}
```

### 21.4 Forgetting to Clear Device Interrupt

```c
/* WRONG — Clears interrupt controller but not device */
/* Result: interrupt fires again immediately after IRET */
static irqreturn_t wrong_isr(int irq, void *dev_id)
{
    process_data(dev_id);
    /* Forgot to write to device's interrupt clear register! */
    return IRQ_HANDLED;
}

/* CORRECT */
static irqreturn_t correct_isr(int irq, void *dev_id)
{
    struct my_device *dev = dev_id;
    u32 status = ioread32(dev->base + STATUS_REG);
    
    /* Clear device interrupt FIRST (before processing, to catch new ones) */
    iowrite32(status, dev->base + STATUS_CLEAR);
    
    process_data(dev);
    return IRQ_HANDLED;
}
```

### 21.5 Missing Memory Barriers for DMA

```c
/* WRONG — CPU may reorder reads, seeing stale DMA data */
static irqreturn_t dma_isr(int irq, void *dev_id)
{
    struct my_device *dev = dev_id;
    
    /* DMA has filled dev->dma_buf, but without barrier,
     * CPU may read stale cache or out-of-order data */
    process_buffer(dev->dma_buf);
    return IRQ_HANDLED;
}

/* CORRECT */
static irqreturn_t dma_isr_correct(int irq, void *dev_id)
{
    struct my_device *dev = dev_id;
    
    /* Ensure all DMA writes are visible before we read the buffer */
    dma_rmb(); /* DMA read memory barrier */
    
    process_buffer(dev->dma_buf);
    return IRQ_HANDLED;
}
```

### 21.6 Calling free_irq from ISR

```c
/* WRONG — Cannot call free_irq from within the ISR itself */
static irqreturn_t bad_self_free_isr(int irq, void *dev_id)
{
    /* This will deadlock — free_irq waits for ISR to complete,
     * but we ARE the ISR! */
    free_irq(irq, dev_id);
    return IRQ_HANDLED;
}

/* CORRECT — Schedule work to do the cleanup */
static irqreturn_t safe_isr(int irq, void *dev_id)
{
    struct my_device *dev = dev_id;
    if (dev->should_stop)
        schedule_work(&dev->cleanup_work);
    return IRQ_HANDLED;
}

static void cleanup_work_fn(struct work_struct *work)
{
    struct my_device *dev = container_of(work, struct my_device, cleanup_work);
    free_irq(dev->irq, dev);  /* Safe from process context */
}
```

### 21.7 Tasklet Deprecation Trap

```c

{
    struct my_device *dev = (struct my_device *)data;

/* NEW approach — tasklet_setup with typed pointer */
```

## 22. ISR in Device Drivers: End-to-End Case Study

### 22.1 Fictional Network Device Driver with Complete ISR Path

```c
// SPDX-License-Identifier: GPL-2.0
/*
 * netdev_isr.c — Complete NIC driver ISR example
 * Demonstrates: DMA ring buffers, NAPI integration,
 *               transmit completion, error handling
 */

#include <linux/module.h>
#include <linux/pci.h>
#include <linux/netdevice.h>
#include <linux/etherdevice.h>
#include <linux/interrupt.h>
#include <linux/dma-mapping.h>
#include <linux/if_ether.h>

#define DRIVER_NAME     "example_nic"
#define RX_RING_SIZE    256
#define TX_RING_SIZE    256
#define MAX_PACKET_SIZE 1522

/* Device register offsets */
#define REG_STATUS      0x00
#define REG_IMR         0x04   /* Interrupt Mask Register */
#define REG_ISR         0x08   /* Interrupt Status Register */
#define REG_RXBASE      0x10   /* RX ring DMA base address */
#define REG_RXPROD      0x18   /* RX producer index */
#define REG_RXCONS      0x1C   /* RX consumer index */
#define REG_TXBASE      0x20   /* TX ring DMA base address */
#define REG_TXPROD      0x28
#define REG_TXCONS      0x2C

/* Interrupt status bits */
#define ISR_RX_COMPLETE  BIT(0)
#define ISR_TX_COMPLETE  BIT(1)
#define ISR_RX_OVERRUN   BIT(2)
#define ISR_LINK_CHANGE  BIT(3)
#define ISR_ERROR        BIT(7)

/* DMA descriptor */
struct nic_desc {
    __le64 dma_addr;
    __le32 length;
    __le32 flags;
} __packed;

#define DESC_FLAG_OWN   BIT(31)  /* 1=owned by hardware, 0=owned by driver */
#define DESC_FLAG_EOP   BIT(30)  /* End of Packet */
#define DESC_FLAG_SOP   BIT(29)  /* Start of Packet */

struct nic_ring {
    struct nic_desc *desc;      /* Descriptor ring (DMA-coherent) */
    dma_addr_t       dma;       /* Physical address of ring */
    struct sk_buff **skbs;      /* SKB shadow ring */
    unsigned int     prod;      /* Producer index */
    unsigned int     cons;      /* Consumer index */
    unsigned int     size;      /* Ring size */
};

struct nic_priv {
    struct pci_dev      *pdev;
    struct net_device   *netdev;
    void __iomem        *base;

    struct nic_ring     rx_ring;
    struct nic_ring     tx_ring;

    struct napi_struct  napi;
    int                 irq;

    /* Statistics */
    struct {
        u64 rx_packets, rx_bytes, rx_errors, rx_dropped;
        u64 tx_packets, tx_bytes, tx_errors;
        u64 irq_count;
    } stats;

    spinlock_t tx_lock;
};

/* ============ RX path ============ */

static int nic_rx_refill(struct nic_priv *priv)
{
    struct nic_ring *ring = &priv->rx_ring;
    unsigned int dirty = ring->prod;
    int count = 0;

    while ((ring->prod - ring->cons) < ring->size - 1) {
        struct sk_buff *skb;
        dma_addr_t dma;
        unsigned int idx = ring->prod & (ring->size - 1);

        /* Allocate SKB for receive buffer */
        skb = netdev_alloc_skb_ip_align(priv->netdev, MAX_PACKET_SIZE);
        if (!skb)
            break;

        /* Map for DMA */
        dma = dma_map_single(&priv->pdev->dev, skb->data,
                              MAX_PACKET_SIZE, DMA_FROM_DEVICE);
        if (dma_mapping_error(&priv->pdev->dev, dma)) {
            dev_kfree_skb_any(skb);
            break;
        }

        ring->skbs[idx] = skb;
        ring->desc[idx].dma_addr = cpu_to_le64(dma);
        ring->desc[idx].length   = cpu_to_le32(MAX_PACKET_SIZE);

        /* DMA write barrier: descriptor must be fully written before
         * giving ownership to hardware */
        dma_wmb();
        ring->desc[idx].flags = cpu_to_le32(DESC_FLAG_OWN);

        ring->prod++;
        count++;
    }

    if (count) {
        /* Tell hardware about new descriptors */
        iowrite32(ring->prod & (ring->size - 1), priv->base + REG_RXPROD);
    }
    return count;
}

/* NAPI poll function — called from softirq context */
static int nic_napi_poll(struct napi_struct *napi, int budget)
{
    struct nic_priv *priv = container_of(napi, struct nic_priv, napi);
    struct nic_ring *ring = &priv->rx_ring;
    int work_done = 0;

    while (work_done < budget) {
        unsigned int idx = ring->cons & (ring->size - 1);
        struct nic_desc *desc = &ring->desc[idx];
        struct sk_buff *skb;
        u32 flags, length;
        dma_addr_t dma;

        /* DMA read barrier: ensure we see hardware's writes */
        dma_rmb();
        flags = le32_to_cpu(desc->flags);

        /* If hardware still owns this descriptor, we're done */
        if (flags & DESC_FLAG_OWN)
            break;

        skb   = ring->skbs[idx];
        dma   = le64_to_cpu(desc->dma_addr);
        length = le32_to_cpu(desc->length);

        /* Unmap DMA before reading data */
        dma_unmap_single(&priv->pdev->dev, dma, MAX_PACKET_SIZE, DMA_FROM_DEVICE);

        if (flags & DESC_FLAG_EOP) {
            /* Complete packet — pass to network stack */
            skb_put(skb, length);
            skb->protocol = eth_type_trans(skb, priv->netdev);
            
            priv->stats.rx_packets++;
            priv->stats.rx_bytes += length;

            /* Hand to network stack — can happen in NAPI/softirq context */
            napi_gro_receive(napi, skb);
        } else {
            /* Error or partial packet */
            dev_kfree_skb_any(skb);
            priv->stats.rx_errors++;
        }

        ring->skbs[idx] = NULL;
        ring->cons++;
        work_done++;
    }

    /* Refill the ring with new SKBs */
    nic_rx_refill(priv);

    /* If we processed less than budget, we're done — exit NAPI */
    if (work_done < budget) {
        napi_complete_done(napi, work_done);
        /* Re-enable RX interrupts */
        u32 imr = ioread32(priv->base + REG_IMR);
        iowrite32(imr | ISR_RX_COMPLETE, priv->base + REG_IMR);
    }

    return work_done;
}

/* ============ TX completion path ============ */

static void nic_tx_complete(struct nic_priv *priv)
{
    struct nic_ring *ring = &priv->tx_ring;
    unsigned long flags;
    u32 hw_cons;

    hw_cons = ioread32(priv->base + REG_TXCONS);

    spin_lock_irqsave(&priv->tx_lock, flags);
    while (ring->cons != hw_cons) {
        unsigned int idx = ring->cons & (ring->size - 1);
        struct sk_buff *skb = ring->skbs[idx];

        if (skb) {
            dma_unmap_single(&priv->pdev->dev,
                              le64_to_cpu(ring->desc[idx].dma_addr),
                              skb->len, DMA_TO_DEVICE);
            priv->stats.tx_packets++;
            priv->stats.tx_bytes += skb->len;
            dev_kfree_skb_irq(skb);  /* Deferred free — safe in softirq */
            ring->skbs[idx] = NULL;
        }
        ring->cons++;
    }

    /* If TX was stopped due to full ring, wake queue */
    if (netif_queue_stopped(priv->netdev) &&
        (ring->prod - ring->cons) < ring->size / 2) {
        netif_wake_queue(priv->netdev);
    }
    spin_unlock_irqrestore(&priv->tx_lock, flags);
}

/* ============ TOP HALF: MSI ISR ============ */

static irqreturn_t nic_isr(int irq, void *dev_id)
{
    struct nic_priv *priv = dev_id;
    u32 status;

    /* Read and clear interrupt status atomically */
    status = ioread32(priv->base + REG_ISR);
    if (!status)
        return IRQ_NONE;  /* Spurious interrupt */

    /* Write-to-clear the ISR register */
    iowrite32(status, priv->base + REG_ISR);

    priv->stats.irq_count++;

    /* TX completion — handle in ISR top half (fast path) */
    if (status & ISR_TX_COMPLETE)
        nic_tx_complete(priv);

    /* RX: schedule NAPI — disable RX interrupt to prevent storm */
    if (status & ISR_RX_COMPLETE) {
        /* Mask RX interrupt — NAPI poll will re-enable when done */
        u32 imr = ioread32(priv->base + REG_IMR);
        iowrite32(imr & ~ISR_RX_COMPLETE, priv->base + REG_IMR);

        /* Schedule NAPI poll (runs in NET_RX_SOFTIRQ) */
        napi_schedule(&priv->napi);
    }

    if (status & ISR_ERROR) {
        /* Schedule work for error recovery (can sleep) */
        schedule_work(&priv->error_work);
    }

    if (status & ISR_LINK_CHANGE) {
        /* Link status changed — schedule work */
        schedule_work(&priv->link_work);
    }

    return IRQ_HANDLED;
}

/* ============ PCI probe ============ */

static int nic_probe(struct pci_dev *pdev, const struct pci_device_id *id)
{
    struct net_device *netdev;
    struct nic_priv *priv;
    int ret;

    /* Enable PCI device */
    ret = pci_enable_device_mem(pdev);
    if (ret) return ret;

    pci_set_master(pdev); /* Enable bus mastering for DMA */

    /* Allocate network device */
    netdev = alloc_etherdev(sizeof(*priv));
    if (!netdev) { ret = -ENOMEM; goto err_alloc; }

    priv = netdev_priv(netdev);
    priv->pdev = pdev;
    priv->netdev = netdev;
    spin_lock_init(&priv->tx_lock);

    /* Map BAR 0 */
    priv->base = pci_iomap(pdev, 0, 0);
    if (!priv->base) { ret = -ENOMEM; goto err_map; }

    /* Set up DMA */
    ret = dma_set_mask_and_coherent(&pdev->dev, DMA_BIT_MASK(64));
    if (ret) goto err_dma;

    /* Enable MSI for a single interrupt vector */
    ret = pci_enable_msi(pdev);
    if (ret) {
        dev_warn(&pdev->dev, "MSI failed, using legacy IRQ\n");
    }
    priv->irq = pdev->irq;

    /* Initialize NAPI */
    netif_napi_add(netdev, &priv->napi, nic_napi_poll, 64);

    /* Register interrupt handler */
    ret = request_irq(priv->irq, nic_isr, 0, DRIVER_NAME, priv);
    if (ret) goto err_irq;

    /* ... register_netdev, etc. ... */
    return 0;

err_irq:
    netif_napi_del(&priv->napi);
err_dma:
    pci_iounmap(pdev, priv->base);
err_map:
    free_netdev(netdev);
err_alloc:
    pci_disable_device(pdev);
    return ret;
}
```

## 23. MSI and MSI-X Interrupts

### 23.1 MSI Advantages

MSI/MSI-X have several advantages over legacy IRQ lines:
- **No IRQ line sharing**: Each vector is dedicated, eliminating shared-IRQ overhead.
- **Multiple vectors per device**: MSI-X allows up to 2048 vectors — one per TX/RX queue in a NIC.
- **CPU affinity per vector**: Each MSI-X vector can be routed to a specific CPU, enabling true RSS (Receive Side Scaling).
- **No spurious interrupts**: Write-based signaling eliminates the phantom interrupt problem.

### 23.2 MSI-X Setup in Driver

```c
#include <linux/pci.h>
#include <linux/interrupt.h>

#define MAX_VECTORS  8

struct nic_priv {
    struct msix_entry msix_entries[MAX_VECTORS];
    int               num_vectors;
    /* ... */
};

static int nic_setup_msix(struct nic_priv *priv)
{
    int nvec, i, ret;

    /* Request as many vectors as possible, minimum 1 */
    nvec = pci_msix_vec_count(priv->pdev);
    nvec = min(nvec, MAX_VECTORS);

    for (i = 0; i < nvec; i++)
        priv->msix_entries[i].entry = i;

    /* Enable MSI-X with the requested number of vectors */
    ret = pci_enable_msix_range(priv->pdev, priv->msix_entries,
                                 1,     /* minimum */
                                 nvec   /* desired */);
    if (ret < 0) {
        dev_err(&priv->pdev->dev, "MSI-X enable failed: %d\n", ret);
        return ret;
    }

    priv->num_vectors = ret;

    /* Register per-queue interrupt handlers */
    for (i = 0; i < priv->num_vectors; i++) {
        snprintf(priv->irq_names[i], sizeof(priv->irq_names[i]),
                 "%s-queue%d", DRIVER_NAME, i);

        ret = request_irq(priv->msix_entries[i].vector,
                          nic_queue_isr,
                          0,
                          priv->irq_names[i],
                          &priv->queues[i]);
        if (ret) {
            while (--i >= 0)
                free_irq(priv->msix_entries[i].vector, &priv->queues[i]);
            pci_disable_msix(priv->pdev);
            return ret;
        }

        /* Pin each MSI-X vector to a specific CPU for RSS */
        irq_set_affinity_hint(priv->msix_entries[i].vector,
                               cpumask_of(i % num_online_cpus()));
    }
    return 0;
}

static void nic_teardown_msix(struct nic_priv *priv)
{
    int i;
    for (i = 0; i < priv->num_vectors; i++) {
        irq_set_affinity_hint(priv->msix_entries[i].vector, NULL);
        free_irq(priv->msix_entries[i].vector, &priv->queues[i]);
    }
    pci_disable_msix(priv->pdev);
}
```

## 24. NAPI and Interrupt Coalescing

### 24.1 The Interrupt Storm Problem

A high-speed NIC at 10 Gbps can receive millions of packets per second. If each packet generates an interrupt, the system would spend 100% of CPU time in ISR context — no time for actual processing.

**NAPI** (New API) solves this with interrupt coalescing:

1. First packet arrives → ISR fires → **disable RX interrupt** → schedule NAPI poll.
2. NAPI poll runs in `NET_RX_SOFTIRQ` → processes up to `budget` packets.
3. If more packets arrive during poll → they're processed in same poll round (no IRQ).
4. When ring is empty → **re-enable RX interrupt** → wait for next packet.

```c
/* NAPI state machine */

/* In ISR: */
if (status & RX_COMPLETE) {
    if (napi_schedule_prep(&priv->napi)) {
        /* Disable RX interrupts */
        iowrite32(IMR & ~RX_IRQ, priv->base + REG_IMR);
        __napi_schedule(&priv->napi);  /* Schedule softirq */
    }
}

/* In NAPI poll: */
static int my_poll(struct napi_struct *napi, int budget)
{
    int done = 0;
    
    while (done < budget && has_more_packets()) {
        process_packet();
        done++;
    }
    
    if (done < budget) {
        /* Processed all available packets — re-arm interrupt */
        napi_complete_done(napi, done);
        iowrite32(IMR | RX_IRQ, priv->base + REG_IMR); /* Re-enable */
    }
    
    return done;  /* Return actual work done */
}
```

### 24.2 Hardware Interrupt Coalescing

Modern NICs support configurable interrupt coalescing in hardware:

```bash
# View current coalescing settings
ethtool -c eth0

# Coalesce Parameters for eth0:
# Adaptive RX: on  TX: off
# rx-usecs: 50          (interrupt after 50µs without a packet)
# rx-frames: 0          (interrupt after N frames, 0=disabled)  
# tx-usecs: 0
# tx-frames: 0
# rx-usecs-irq: 0       (usecs to coalesce when IRQ is being served)
# rx-frames-irq: 0

# Set to interrupt every 100µs or 64 frames (whichever comes first)
ethtool -C eth0 rx-usecs 100 rx-frames 64
```

In the driver, this is implemented via:

```c
static int nic_set_coalesce(struct net_device *netdev,
                             struct ethtool_coalesce *ec,
                             struct kernel_ethtool_coalesce *kec,
                             struct netlink_ext_ack *extack)
{
    struct nic_priv *priv = netdev_priv(netdev);
    
    /* Validate */
    if (ec->rx_coalesce_usecs > 1000)
        return -EINVAL;
    
    /* Program hardware coalescing registers */
    iowrite32(ec->rx_coalesce_usecs, priv->base + REG_COALESCE_TIME);
    iowrite32(ec->rx_max_coalesced_frames, priv->base + REG_COALESCE_CNT);
    
    return 0;
}
```

## 25. Glossary

| Term | Definition |
|------|------------|
| **APIC** | Advanced Programmable Interrupt Controller. Modern x86 interrupt controller replacing the 8259A PIC. |
| **Bottom Half** | Deferred interrupt processing: softirq, tasklet, workqueue, or threaded IRQ. Runs outside atomic context. |
| **CLI/STI** | x86 instructions to clear/set the Interrupt Flag (IF), disabling/enabling maskable interrupts. |
| **cmwq** | Concurrency-Managed Workqueue. Linux's workqueue implementation that manages worker thread pools dynamically. |
| **DMA** | Direct Memory Access. Allows hardware to read/write RAM without CPU involvement. Interrupt fires when complete. |
| **DPL** | Descriptor Privilege Level. Gate descriptor field that controls from which privilege level the gate can be invoked. |
| **Edge-Triggered** | Interrupt asserted on a signal transition (low→high or high→low). |
| **EOI** | End of Interrupt. Signal sent to interrupt controller after ISR completes. Required for proper interrupt delivery. |
| **GIC** | Generic Interrupt Controller. ARM's standard interrupt controller architecture (v1-v4). |
| **GFP_ATOMIC** | Get Free Pages with Atomic flag. Memory allocation that never sleeps; for use in interrupt context. |
| **hardirq** | The top-half ISR — runs directly in interrupt context with interrupts disabled (or at least the current IRQ masked). |
| **IDT** | Interrupt Descriptor Table. x86 table of 256 gate descriptors that maps interrupt vectors to handler addresses. |
| **IDTR** | IDT Register. x86 register (loaded via LIDT) that stores the base and limit of the IDT. |
| **IPI** | Inter-Processor Interrupt. An interrupt sent from one CPU to another via the APIC ICR. |
| **IRQ** | Interrupt ReQuest. A signal line used by a device to request CPU attention. |
| **irq_chip** | Linux kernel structure providing the interface to an interrupt controller chip. |
| **irq_desc** | Per-interrupt descriptor in the Linux kernel. Central structure of the IRQ subsystem. |
| **irq_domain** | Linux mapping between hardware interrupt numbers and Linux virtual IRQ numbers. |
| **IRET** | Interrupt Return. x86 instruction that returns from an ISR, restoring saved RFLAGS, CS, RIP. |
| **ISR** | Interrupt Service Routine. The function called when a specific interrupt is delivered. |
| **IST** | Interrupt Stack Table. x86-64 mechanism for switching to a dedicated stack on interrupt entry. |
| **LAPIC** | Local APIC. Per-CPU interrupt controller responsible for receiving and acknowledging interrupts. |
| **Level-Triggered** | Interrupt asserted as long as the signal line is held in the active state. |
| **LPI** | Locality-specific Peripheral Interrupt. GICv3+ message-based interrupts, analogous to MSI-X. |
| **MSI** | Message Signaled Interrupt. PCIe interrupt mechanism using memory writes instead of dedicated wires. |
| **MSI-X** | Extended MSI. Supports up to 2048 independent interrupt vectors per device. |
| **NAPI** | New API. Linux network interrupt coalescing mechanism that combines polling and interrupts. |
| **NMI** | Non-Maskable Interrupt. An interrupt that cannot be disabled via the IF flag. Used for hardware faults and watchdogs. |
| **NUMA** | Non-Uniform Memory Access. Memory topology where access latency depends on which CPU accesses which memory. |
| **PIC** | Programmable Interrupt Controller. The legacy 8259A chip providing 15 IRQ lines in older PC systems. |
| **PPR** | Processor Priority Register. LAPIC register indicating current interrupt priority threshold. |
| **PREEMPT_RT** | Real-Time Preemption patch set that makes Linux a hard real-time OS by threading all IRQs and making spinlocks sleepable. |
| **Priority Inheritance** | RT mechanism where a low-priority task inherits the priority of a high-priority task blocked on a resource it holds. |
| **pt_regs** | Process registers. Kernel structure capturing the full CPU register state at interrupt entry. |
| **RCU** | Read-Copy-Update. A synchronization mechanism allowing lock-free reads, safe in interrupt context. |
| **RTIC** | Real-Time Interrupt-driven Concurrency. A Rust framework for safe, concurrent bare-metal embedded systems. |
| **softirq** | Software interrupt. A deferred work mechanism in Linux that runs with interrupts enabled but cannot sleep. |
| **SPI** | Shared Peripheral Interrupt (GIC). Interrupts from peripheral devices shared among CPUs. |
| **SGI** | Software Generated Interrupt (GIC). Used for IPIs on ARM systems. |
| **Tasklet** | A bottom-half mechanism built on softirqs. Serialized (won't run concurrently with itself) and dynamically created. |
| **Top Half** | The ISR itself — the minimum work done synchronously when the interrupt fires. |
| **TPR** | Task Priority Register. LAPIC register setting the minimum interrupt priority the CPU will accept. |
| **UIO** | Userspace I/O. Linux framework enabling device drivers (including interrupt handling) to run in user space. |
| **Vector** | An 8-bit number (0–255) indexing into the IDT. Each interrupt source is assigned a vector number. |
| **Workqueue** | A bottom-half mechanism where work items run in process context (can sleep, allocate with GFP_KERNEL). |
| **x2APIC** | Extended APIC. APIC variant accessed via MSRs instead of memory-mapped I/O, supporting >255 CPU IDs. |
